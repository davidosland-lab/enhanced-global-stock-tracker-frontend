TRADE MODE CONFIGURATION
================================================================================

Default: "strict" mode (conservative)
--------------------------------------
- Only trade BUY/STRONG_BUY signals
- HOLD signals always blocked
- Safe, respects ML model predictions

How to Enable "confidence_based" Mode:
---------------------------------------

Method 1 - Dashboard UI (Easiest):
  1. Start dashboard
  2. Find "Trade Mode" section (below Stop Loss)
  3. Select "Confidence-Based (Allow high-confidence HOLD)"
  4. Start trading (config saved automatically)

Method 2 - Edit config/config.json:
  {
    "paper_trading": {
      "trade_mode": "confidence_based",
      "min_confidence": 53.0,
      "hold_override_min_score": 60.0
    }
  }

Expected Behavior:
------------------
Strict Mode Logs:
  [BLOCK] CNA.L: Signal not actionable (strict mode)
  [BLOCK] BA.L: Signal not actionable (strict mode)

Confidence_Based Mode Logs:
  [HOLD→BUY] CNA.L: Overriding HOLD signal - score=70.7, conf=53.6%
  [HOLD→BUY] BA.L: Overriding HOLD signal - score=60.7, conf=53.6%

TROUBLESHOOTING
================================================================================

Common Issues & Solutions:
--------------------------

1. "ModuleNotFoundError: No module named 'pandas'"
   → Run INSTALL_FIRST_TIME.bat again

2. "Address already in use" (port 8050)
   → Kill process or change port

3. HOLD signals still blocked in confidence_based mode
   → Check config.json: "trade_mode": "confidence_based"
   → Check logs for "[HOLD→BUY]" messages

4. Too many HOLD signals trading
   → Increase hold_override_min_score (60 → 65 or 70)
   → Or switch to "strict" mode

5. UnicodeEncodeError in logs
   → Should be fixed in v193.11.6.15
   → Re-extract package if using older version

6. US pipeline AttributeError
   → Should be fixed in v193.11.6.13
   → Check VERSION.json shows correct version
