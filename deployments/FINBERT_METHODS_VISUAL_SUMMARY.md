## 🔑 Key Insights

### 1. Sentiment is Only 15%
```
┌─────────────────────────────────┐
│ News sentiment:     15% weight  │
│ Everything else:    85% weight  │
└─────────────────────────────────┘

News provides CONTEXT, not DECISION
```

### 2. Price Action Beats News
```
News says: "AAPL faces regulation" → Negative
Price says: "Buyers stepping in"   → Rising
Volume says: "Strong conviction"    → High

→ Trust price action over headlines
```

### 3. Ensemble Reduces False Signals
```
Single Model:
├─ One bad signal → Wrong decision
└─ No validation → High risk

Ensemble:
├─ 5 models vote → Consensus needed
├─ Volume confirms → Extra validation
└─ Lower risk → Higher accuracy
```

### 4. Volume is the Validator
```
High Volume + Signal  = ✅ Trust it
Low Volume + Signal   = ⚠️ Be cautious
No Volume + Signal    = ❌ Skip it
```

## 🔧 Technical Specifications

### System Requirements:
- **Python**: 3.8+
- **TensorFlow**: 2.x (for LSTM)
- **PyTorch**: 1.x (for FinBERT sentiment)
- **Libraries**: ta, pandas, numpy, scikit-learn

### Model Files:
```
finbert_v4.4.4/
├── app_finbert_v4_dev.py          ← Main ensemble
├── models/
│   ├── lstm_predictor.py          ← LSTM model (45%)
│   ├── finbert_sentiment.py       ← Sentiment (15%)
│   ├── news_sentiment_real.py     ← News scraping
│   └── prediction_manager.py      ← Orchestration
└── config_dev.py                  ← Configuration
```

### Key Functions:
- `get_ensemble_prediction()` - Main entry point
- `get_lstm_prediction()` - LSTM analysis (45%)
- `simple_prediction()` - Trend analysis (25%)
- `technical_prediction()` - Technical analysis (15%)
- `sentiment_prediction()` - Sentiment analysis (15%)
- `analyze_volume()` - Volume validation (adjustment)
