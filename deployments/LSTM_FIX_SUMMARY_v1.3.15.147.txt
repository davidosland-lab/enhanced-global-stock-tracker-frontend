When you add a directory to sys.path, imports are relative to THAT directory!

❌ WRONG:
sys.path.insert(0, '/path/to/models')
from models.something import ...  # Looks for /path/to/models/models/something.py

✅ CORRECT:
sys.path.insert(0, '/path/to/models')
from something import ...  # Looks for /path/to/models/something.py

The diagnostic logging (v1.3.15.146) was crucial - it showed the file existed
at the path we checked, but the import was adding an extra "models/" level!
