====================================================================================
PROBLEM STATEMENT
====================================================================================

Windows users were experiencing UnicodeEncodeError during pipeline execution:

  UnicodeEncodeError: 'charmap' codec can't encode character '\u2713' in 
  position 52: character maps to <undefined>
  
  File: C:\Python312\Lib\logging\__init__.py", line 1203, in emit
  
This error occurred when logger calls contained Unicode special characters 
(checkmarks ✓, arrows →, emojis 🚨) that are not available in Windows cp1252 
console encoding.

Despite v193.11.1 adding UTF-8 encoding to logging handlers, certain Unicode 
characters cannot be represented in cp1252 and cause Python's logging module 
to fail when writing to stdout.

====================================================================================
ROOT CAUSE ANALYSIS
====================================================================================

1. SCOPE: 27 Python files contained Unicode characters in logger calls
2. CHARACTERS: 25+ different Unicode symbols used across the codebase
3. VOLUME: 49 total replacements needed across all files
4. IMPACT: All three pipelines (AU, US, UK) affected during runtime

Most Critical Files:
  - finbert_v4.4.4/models/lstm_predictor.py (13 occurrences of ❌)
  - pipelines/models/screening/spi_monitor.py (9 occurrences across 5 symbols)
  - pipelines/models/screening/overnight_pipeline.py (5 occurrences)

PREVENTION GUIDELINES
====================================================================================

For ALL future development, follow these strict rules:

❌ NEVER DO THIS:
  logger.info(f"✓ Success for {symbol}")
  logger.error(f"❌ Failed to process {data}")
  logger.warning(f"⚠ Check this value: {value}")

✅ ALWAYS DO THIS:
  logger.info(f"[OK] Success for {symbol}")
  logger.error(f"[ERROR] Failed to process {data}")
  logger.warning(f"[!] Check this value: {value}")

APPROVED ASCII TAGS:
  [OK]     - Success, completion, affirmative
  [X]      - Failure, negative result
  [!]      - Warning, caution
  [ERROR]  - Error condition
  [ALERT]  - High-priority warning
  [LOCKED] - Security/access restriction
  [CACHE]  - Cached data usage
  [*]      - Processing, working
  [CHART]  - Data visualization
  [STOP]   - Stopped, halted
  [TARGET] - Goal, objective
  [BOOST]  - Enhancement, amplification
  [LIST]   - Enumeration, collection
  [TOOL]   - Utility, helper function
  [AI]     - Artificial intelligence component
  [DOC]    - Documentation
  [PASS]   - Test passed

APPROVED ASCII SYMBOLS:
  ->       - Instead of →
  >=       - Instead of ≥
  <=       - Instead of ≤
  x        - Instead of ×
  GBP      - Instead of £
  USD      - Instead of $
  [US]     - Instead of 🇺🇸
  [AU]     - Instead of 🇦🇺
  [UK]     - Instead of 🇬🇧

AUTOMATIC CLEANUP:
  If you accidentally introduce Unicode in logger calls, run:
  python HOTFIX_v193.11.5_REMOVE_ALL_UNICODE_LOGGING.py
