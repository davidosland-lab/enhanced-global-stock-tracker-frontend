USER QUESTION
================================================================================

"Is rigidly sticking to the HOLD signal overriding the setting we had that 
trades could start at 53% confidence?"

ANSWER: YES - The system WAS rigidly sticking to HOLD signals. Signal type 
acted as a hard gate that blocked HOLD signals before confidence was even 
checked. This is now CONFIGURABLE via the new 'trade_mode' setting.

================================================================================
PROBLEM ANALYSIS
================================================================================

Example from User's Log:
- CNA.L: Score 70.7/100, Signal HOLD, Confidence 53.6%
- BA.L: Score 60.7/100, Signal HOLD, Confidence 53.6%

Result: Both stocks BLOCKED despite meeting 53% confidence threshold

Root Cause:
The system had TWO filtering stages that worked sequentially:

STAGE 1: Signal Type Filter (HARD GATE)
  - Location: _evaluate_pipeline_recommendation() line 612-645
  - Logic: if signal not in ['BUY', 'STRONG_BUY', 'SELL', 'STRONG_SELL']:
             return False, 0, "Signal not actionable"
  - HOLD signals rejected HERE, before Stage 2

STAGE 2: Confidence Filter
  - Location: Various places, including line 627
  - Logic: if confidence < min_confidence: reject
  - NEVER REACHED for HOLD signals (already blocked in Stage 1)

Impact:
- Confidence threshold (53%) was IRRELEVANT for HOLD signals
- High-quality stocks like CNA.L (score 70.7) blocked
- User's confidence setting was bypassed by signal type filter

================================================================================
SOLUTION - HYBRID TRADE MODE
================================================================================

Added configurable 'trade_mode' parameter with two options:

┌─────────────────────────────────────────────────────────────────────────┐
│ MODE 1: "strict" (DEFAULT)                                              │
├─────────────────────────────────────────────────────────────────────────┤
│ Behavior: Original conservative logic                                   │
│ - Only trade explicit BUY/STRONG_BUY signals for entries               │
│ - Only trade SELL/STRONG_SELL signals for exits                        │
│ - HOLD signals always blocked regardless of confidence or score        │
│                                                                          │
│ Philosophy: "Respect ML model's directional assessment"                │
│                                                                          │
│ Pros: Conservative, respects model uncertainty                          │
│ Cons: Misses high-quality neutral stocks                                │
├─────────────────────────────────────────────────────────────────────────┤
│ Example Results:                                                         │
│ - CNA.L (70.7, HOLD, 53.6%): BLOCKED ❌                                │
│ - BA.L (60.7, HOLD, 53.6%): BLOCKED ❌                                 │
│ - Low stock (35, HOLD, 55%): BLOCKED ❌                                 │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ MODE 2: "confidence_based" (NEW)                                        │
├─────────────────────────────────────────────────────────────────────────┤
│ Behavior: Confidence threshold as primary filter                        │
│ - Check confidence FIRST (e.g., ≥53%)                                  │
│ - Use score as directional bias indicator                               │
│ - Allow high-quality HOLD signals to trade                              │
│                                                                          │
│ Rules:                                                                   │
│ 1. High-quality HOLD (score ≥60, sentiment ≥45, conf ≥53%) → BUY      │
│ 2. Low-quality HOLD (score ≤40) + open position → SELL                 │
│ 3. Neutral HOLD (40 < score < 60) → Still BLOCKED                      │
│ 4. Explicit BUY/SELL signals: Same rules as strict mode                │
│                                                                          │
│ Philosophy: "If confidence is high, trade in direction of score bias"  │
│                                                                          │
│ Pros: Confidence threshold meaningful, trades high-quality stocks        │
│ Cons: May override model's neutral assessment                           │
├─────────────────────────────────────────────────────────────────────────┤
│ Example Results:                                                         │
│ - CNA.L (70.7, HOLD, 53.6%): TRADED AS BUY ✅ (score≥60, conf≥53)     │
│ - BA.L (60.7, HOLD, 53.6%): TRADED AS BUY ✅ (score≥60, conf≥53)      │
│ - Low stock (35, HOLD, 55%): TRADED AS SELL ✅ (if position exists)    │
│ - Neutral (50, HOLD, 60%): BLOCKED ❌ (40<score<60, truly neutral)     │
└─────────────────────────────────────────────────────────────────────────┘

Rewrote _evaluate_pipeline_recommendation() method (lines 597-706):

def _evaluate_pipeline_recommendation(self, recommendation):
    # Get trade mode from config (default 'strict')
    trade_mode = self.config.get('paper_trading', {}).get('trade_mode', 'strict')
    
    if trade_mode == 'strict':
        # ORIGINAL BEHAVIOR
        if signal in ['BUY', 'STRONG_BUY']:
            # Check score, sentiment, calculate confidence
            return True, confidence, reason
        elif signal in ['SELL', 'STRONG_SELL']:
            # Check position, score
            return True, confidence, reason
        else:
            # HOLD or other → BLOCK
            return False, 0, f"Signal {signal} not actionable (strict mode)"
    
    elif trade_mode == 'confidence_based':
        # NEW BEHAVIOR
        confidence = (score * 0.7 + sentiment * 0.3)
        
        # Check confidence FIRST
        if confidence < min_confidence:
            return False, 0, f"Confidence {confidence:.1f}% < {min_confidence}%"
        
        # Explicit BUY/SELL: same as strict
        if signal in ['BUY', 'STRONG_BUY']:
            return True, confidence, reason
        elif signal in ['SELL', 'STRONG_SELL']:
            return True, confidence, reason
        
        # HOLD signal: use score-based bias
        elif signal == 'HOLD':
            if score >= 60.0 and sentiment >= 45.0:
                # High-quality HOLD → Trade as BUY
                logger.info(f"[HOLD→BUY] {symbol}: score={score}, conf={confidence}")
                return True, confidence, f"Pipeline BUY (HOLD override)"
            
            elif score <= 40.0 and symbol in positions:
                # Low-quality HOLD + position → Trade as SELL
                logger.info(f"[HOLD→SELL] {symbol}: score={score}, conf={confidence}")
                return True, confidence, f"Pipeline SELL (HOLD override)"
            
            else:
                # Truly neutral HOLD (40 < score < 60)
                return False, 0, f"Signal HOLD with neutral score {score}"
