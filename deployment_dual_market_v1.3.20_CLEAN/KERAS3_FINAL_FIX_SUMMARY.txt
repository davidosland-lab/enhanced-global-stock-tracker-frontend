================================================================================
WHY .keras FORMAT IS BETTER
================================================================================

Keras 3 Team Recommendation:
  "We recommend using the native Keras format, e.g. model.save('model.keras')"

Benefits:
  ✓ Native Keras 3 format (not legacy)
  ✓ Better cross-framework support
  ✓ More efficient storage
  ✓ No deprecation warnings
  ✓ Future-proof (won't be removed)
  ✓ Automatic format inference
  ✓ Simpler code (no save_format parameter)

.h5 format issues:
  ✗ Legacy format from Keras 2
  ✗ Generates deprecation warnings
  ✗ Requires save_format='h5' parameter
  ✗ May be removed in future
  ✗ Less efficient
