WHAT THIS FIX SOLVES
================================================================================

PROBLEM:
--------
Your system has TensorFlow 2.20.0 with Keras 3.11.3 installed.

Keras 3 introduced a BREAKING CHANGE:
  • Default save format changed from .h5 to .keras
  • Saving to .h5 requires explicit: save_format='h5'
  • Without this, model.save() SILENTLY FAILS

SYMPTOMS YOU EXPERIENCED:
-------------------------
  ✗ Pipeline trains models successfully
  ✗ Only JSON metadata files created (lstm_A2M.AX_metadata.json)
  ✗ NO .h5 model files saved (A2M.AX_lstm_model.h5 missing)
  ✗ Pipeline retrains all 139 models every run
  ✗ 2+ hours wasted per run
  ✗ No efficiency gains from caching

ROOT CAUSE:
----------
File: finbert_v4.4.4/models/lstm_predictor.py
Line 510: self.model.save(self.model_path)
Missing: save_format='h5' parameter

THE FIX:
-------
Adds save_format='h5' parameter to model.save() call:

  OLD: self.model.save(self.model_path)
  NEW: self.model.save(self.model_path, save_format='h5')

Plus:
  • Keras 2.x fallback compatibility
  • Enhanced error logging
  • Detailed traceback on failures
