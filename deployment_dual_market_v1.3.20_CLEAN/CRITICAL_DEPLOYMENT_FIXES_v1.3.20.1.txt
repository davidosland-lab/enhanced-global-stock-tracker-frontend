FIX #2: HMM Model Covariance Error
----------------------------------
ERROR: "Error fitting HMM model: 'covars' must be symmetric, positive-definite"
LOCATION: models/screening/us_market_regime_engine.py, fit_model()
FIX: Added StandardScaler normalization and changed covariance type
STATUS: ✅ FIXED

Changes:
- Added StandardScaler to normalize features before HMM fitting
- Changed covariance_type from "full" to "diag" for better stability
- Added small regularization noise (1e-6) to ensure positive-definiteness
- Stored scaler for consistent prediction scaling
- Added graceful fallback if HMM still fails

Result: HMM model should now fit successfully without covariance errors.
System still has fallback regime detection if HMM fails.
