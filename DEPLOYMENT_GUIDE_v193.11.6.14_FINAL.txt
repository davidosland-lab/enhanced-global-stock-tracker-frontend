🎯 WHAT'S NEW IN v193.11.6.14
================================================================================

🔴 **CRITICAL BUG FIX: HOLD Signals Were Being Traded**

**Problem Reported by User:**
  - CNA.L (HOLD signal) was being traded
  - BA.L (HOLD signal) was being traded
  - Question: "Why was BA.L traded when it has HOLD signal?"

**Root Cause:**
  - Inverted logic in should_allow_trade() method
  - Line 1335 returned True (allow) instead of False (block) for non-BUY signals
  - ALL HOLD signals were being traded as if they were BUY signals

**Solution:**
  ✅ Fixed inverted logic: return True → return False
  ✅ HOLD signals now properly BLOCKED from trading
  ✅ Only BUY and STRONG_BUY trigger new positions
  ✅ Added [BLOCK] log messages for visibility

**Impact:**
  - No more unwanted trades on HOLD signals
  - Capital allocated only to true BUY opportunities
  - System behavior now matches user expectations
  - "Load Top 50" loads watchlist but only trades BUY signals

**Issue: HOLD signals still being traded**
-------------------------------------------
Solution:
1. Verify you're running v193.11.6.14:
   ```
   cat VERSION.json | grep version
   ```
   Should show: "version": "1.3.15.193.11.6.14"

2. Restart dashboard completely:
   ```
   # Kill existing process
   pkill -f unified_trading_dashboard
   
   # Start fresh
   python core/unified_trading_dashboard.py
   ```

3. Check code fix in paper_trading_coordinator.py line 1334-1336:
   ```
   if not is_buy_signal:
       return False, 0.0, reason  # ← Should be FALSE, not TRUE
   ```

**Issue: US Pipeline AttributeError**
--------------------------------------
Solution:
1. Verify event_risk_guard.py has get_latest_regime() method (line 565-589)
2. Check VERSION.json shows v193.11.6.13 or later
3. Restart US pipeline

**Issue: ModuleNotFoundError**
-------------------------------
Solution:
1. Run INSTALL_FIRST_TIME.bat again
2. Manually install missing package:
   ```
   pip install <package-name>
   ```

**Issue: Dashboard won't start**
---------------------------------
Solution:
1. Check if port 8050 is in use:
   ```
   netstat -ano | findstr :8050  (Windows)
   lsof -i :8050                  (Linux/Mac)
   ```
2. Kill existing process or change port in code
