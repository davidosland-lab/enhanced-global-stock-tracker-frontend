================================================================================
ROOT CAUSE
================================================================================

FinBERT requires TWO deep learning libraries that are NOT installed:

1. **transformers** (HuggingFace Transformers)
   - Size: ~500 MB
   - Purpose: Loads pre-trained BERT models
   - Required for: FinBERT neural network

2. **torch** (PyTorch)
   - Size: ~800 MB to 2 GB (depending on CPU/GPU version)
   - Purpose: Deep learning framework
   - Required for: Running neural networks

These libraries were NOT included because:
- Very large download size (1-2.5 GB total)
- Optional dependency (fallback mode works without them)
- Installation can take 5-15 minutes on slow connections
- Some environments may not have enough disk space

================================================================================
FALLBACK MODE vs FULL MODE
================================================================================

Current: FALLBACK MODE (Keyword-Based)
---------------------------------------
Accuracy: 60-70%
Method: Keyword matching
Speed: Very fast
Pros:
- ✅ No installation required
- ✅ Works immediately
- ✅ Small memory footprint
- ✅ Fast execution

Cons:
- ❌ Lower accuracy (60-70%)
- ❌ Misses context and nuance
- ❌ Simple pattern matching
- ❌ Can be fooled by sarcasm/negation

Example:
"Stock rises despite weak earnings" 
- Fallback: POSITIVE (sees "rises")
- FinBERT: NEGATIVE (understands "despite" negates positivity)


Full: FINBERT MODE (Neural Network)
------------------------------------
Accuracy: 75-85%
Method: Deep learning transformer model
Speed: Slower (but cached after first load)
Pros:
- ✅ Higher accuracy (75-85%)
- ✅ Understands context and nuance
- ✅ Pre-trained on 1.8M financial texts
- ✅ Handles complex sentences

Cons:
- ❌ Requires 1-2.5 GB download
- ❌ First load takes 10-30 seconds
- ❌ Higher memory usage (~500 MB)
- ❌ Installation complexity

Example:
"Stock rises despite weak earnings"
- FinBERT: NEGATIVE (understands the overall bearish context)

================================================================================
WHY FALLBACK MODE EXISTS
================================================================================

Design Decision: FinBERT was designed with fallback mode to ensure:

1. **Immediate Functionality**
   - System works out-of-the-box without complex setup
   - No failed imports blocking execution
   - Graceful degradation

2. **Environment Flexibility**
   - Works in resource-constrained environments
   - No GPU required
   - Smaller disk space requirements

3. **Optional Enhancement**
   - Users can choose accuracy vs complexity trade-off
   - Advanced users install deep learning libraries
   - Beginners use fallback mode

4. **Backwards Compatibility**
   - Older systems without deep learning support still work
   - No breaking changes to existing setups

================================================================================
INSTALLATION TROUBLESHOOTING
================================================================================

Problem 1: "ERROR: Could not find a version that satisfies the requirement torch"
Solution: Update pip first
```bash
pip install --upgrade pip
pip install transformers torch
```

Problem 2: "Out of memory" during installation
Solution: Install separately to reduce memory pressure
```bash
pip install torch
pip install transformers
```

Problem 3: Installation takes too long (>30 minutes)
Solution: Use a faster mirror or pre-built wheels
```bash
pip install --default-timeout=1000 transformers torch
```

Problem 4: "No matching distribution found"
Solution: Check Python version (requires Python 3.8+)
```bash
python --version
# If < 3.8, upgrade Python first
```

Problem 5: SSL certificate error
Solution: Use trusted host flag
```bash
pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org transformers torch
```
