================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Port already in use"
SOLUTION: 
  Option A: Kill existing process
    Windows: 
      netstat -ano | findstr :5001
      taskkill /PID <PID> /F
    
    Mac/Linux:
      lsof -ti:5001 | xargs kill -9
  
  Option B: Use different port
    Set environment variable:
      export FLASK_PORT=5002
    Then start server


PROBLEM: "ModuleNotFoundError: No module named 'flask'"
SOLUTION:
  Install dependencies:
    pip install flask
  
  Or install all:
    pip install -r requirements.txt


PROBLEM: "TensorFlow not installed"
SOLUTION:
  This is a warning, not an error. Server still works!
  
  To enable LSTM training:
    pip install tensorflow keras
  
  Can run without it (uses simpler predictions)


PROBLEM: "FinBERT libraries not available"
SOLUTION:
  This is a warning, not an error. Server still works!
  
  To enable sentiment analysis:
    pip install transformers torch
  
  Can run without it (uses fallback sentiment)


PROBLEM: "Port 5001 unreachable in browser"
SOLUTION:
  1. Check server is running (terminal should show "Running on...")
  2. Try http://127.0.0.1:5001 instead of localhost
  3. Check firewall settings
  4. Try different port (see "Port already in use" above)


PROBLEM: Paper Trading modal won't open
SOLUTION:
  1. Open browser console (F12)
  2. Check for JavaScript errors
  3. Verify /api/trading/account endpoint:
       curl http://localhost:5001/api/trading/account
  4. Restart server


PROBLEM: Predictions show low confidence
SOLUTION:
  Normal! Causes:
  - No LSTM model trained for that stock
  - Market closed (after-hours = lower confidence)
  - Conflicting technical indicators
  - Low trading volume
  
  To improve:
  - Train LSTM model for the stock
  - Use during market hours
  - Check multiple stocks


PROBLEM: Charts not displaying
SOLUTION:
  1. Check internet connection (loads Chart.js from CDN)
  2. Check browser console for errors
  3. Try different browser
  4. Clear browser cache


PROBLEM: Training fails with memory error
SOLUTION:
  Reduce epochs:
  - Change from 50 to 20 epochs
  - Or train fewer stocks at a time
  
  Close other applications to free memory

UNINSTALLATION
================================================================================

To remove FinBERT v4.4:

1. Stop the server (Ctrl+C in terminal)

2. Deactivate virtual environment (if used):
     deactivate

3. Delete installation directory

4. Remove database (optional):
     rm trading.db
     rm predictions.db
     rm backtest_cache.db

5. Uninstall Python packages (optional):
     pip uninstall flask yfinance pandas numpy ta tensorflow transformers
