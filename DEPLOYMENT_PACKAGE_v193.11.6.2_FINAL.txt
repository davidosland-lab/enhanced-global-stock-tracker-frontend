📞 SUPPORT & TROUBLESHOOTING
================================================================================

ISSUE: "unrecognized arguments: --mode production"
  ✅ SOLVED in v193.11.6.2
  Action: Upgrade to v193.11.6.2

ISSUE: ModuleNotFoundError for pandas, numpy, tensorflow, etc.
  Cause: Dependencies not installed
  Fix: Run INSTALL_FIRST_TIME.bat (10-15 min)
       Then run INSTALL_COMPLETE_v193.bat (5-10 min)

ISSUE: Web Control Center won't start
  Causes:
    - Flask not installed
    - Port 5000 in use
    - Windows Firewall blocking
  Fixes:
    - pip install flask flask-cors
    - Change port in web_control_center.py
    - Allow port 5000 in Windows Firewall

ISSUE: Dashboard iframe is blank
  Causes:
    - Dashboard still starting (takes ~30 seconds)
    - Port 8050 in use
    - Browser cache
  Fixes:
    - Wait 30 seconds after clicking "Launch Dashboard"
    - Check no other app using port 8050
    - Refresh page (F5)

ISSUE: Cannot access from phone/tablet
  Causes:
    - Windows Firewall blocking port 5000
    - Different WiFi networks
    - Wrong IP address
  Fixes:
    - Allow port 5000 in Windows Firewall
    - Ensure PC and phone on same WiFi
    - Use PC's IP: http://192.168.1.XXX:5000
    - Check PC IP with: ipconfig

ISSUE: Pipelines not starting (even in v193.11.6.2)
  Diagnostic:
    - Check VERSION.json shows v1.3.15.193.11.6.2
    - Check terminal output for specific errors
    - Test yfinance: python -c "import yfinance"
    - Check market data access (not rate-limited)
  Fixes:
    - Verify correct version extracted
    - Check dependencies installed
    - Wait and retry (if rate-limited)
    - Check internet connection

ISSUE: FinBERT not working
  Causes:
    - FinBERT dependencies not installed
    - Model files not downloaded
    - GPU/CPU backend issues
  Fixes:
    - Run INSTALL_FINBERT.bat (downloads ~2.5 GB)
    - Takes 10-15 minutes
    - Requires ~3 GB disk space
    - Sets PyTorch backend automatically

ISSUE: Slow performance / high memory usage
  Causes:
    - All three pipelines running simultaneously
    - Large historical data cache
    - Too many browser tabs
  Fixes:
    - Run only one pipeline at a time
    - Clear cache: delete data/ folders
    - Close unused tabs
    - Increase virtual memory

ISSUE: yfinance download failures
  Causes:
    - Yahoo Finance rate limiting
    - Network issues
    - Invalid ticker symbols
  Fixes:
    - Wait 5-10 minutes between scans
    - Check internet connection
    - Verify ticker symbols valid
    - Use --symbols flag for targeted scans
