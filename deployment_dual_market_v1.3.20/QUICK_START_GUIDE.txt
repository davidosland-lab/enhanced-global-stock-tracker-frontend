================================================================================
  UNDERSTANDING REGIME ANALYSIS
================================================================================

VOLATILITY STATES:
üü¢ Low Volatility
   - Stable market conditions
   - Low crash risk (typically <15%)
   - Recommendation: Accumulate quality stocks
   - Trading: Favorable for long positions

üü° Medium Volatility
   - Moderate market conditions
   - Medium crash risk (15-30%)
   - Recommendation: Monitor positions, use stop-losses
   - Trading: Increased caution advised

üî¥ High Volatility
   - High volatility environment
   - High crash risk (>30%)
   - Recommendation: Reduce exposure, protect capital
   - Trading: Consider defensive positions

CRASH RISK SCORE:
- Percentage indicating probability of market crash
- Based on HMM (Hidden Markov Model) analysis
- Uses historical volatility patterns
- Updates with each screening run

================================================================================
  TROUBLESHOOTING
================================================================================

PROBLEM: "Module not found" errors
SOLUTION: 
1. Run CHECK_INSTALLATION.bat
2. Ensure setup_paths.py is in root directory
3. Read TROUBLESHOOTING_IMPORTS.md

PROBLEM: Reports don't show regime section
SOLUTION:
1. Check screening_config.json has regime_detection_enabled: true
2. Run pipeline at least once
3. Check logs for regime detection errors

PROBLEM: Web UI shows "not available"
SOLUTION:
1. Run screening first to generate data
2. Check reports directory has JSON files
3. Check web UI logs for errors

PROBLEM: Import warnings about LSTM/FinBERT
SOLUTION:
- These are EXPECTED warnings for optional components
- System works fine without them
- Read TROUBLESHOOTING_IMPORTS.md for details

PROBLEM: US market not working
SOLUTION:
1. Check us_sectors.json exists
2. Run: python run_screening.py --market us --verbose
3. Check logs/screening/us/*.log
4. Ensure internet connection for data fetching
