LSTM PENALTY SYSTEM EXPLAINED
================================================================================

WHAT IS THE PENALTY SYSTEM?
---------------------------
When the trading system generates signals, it uses 5 components:
1. FinBERT Sentiment (25% weight)
2. **LSTM Neural Network (25% weight)**  ← Requires pre-trained model
3. Technical Analysis (25% weight)
4. Momentum Analysis (15% weight)
5. Volume Analysis (10% weight)

**IF LSTM MODEL IS MISSING:**
- System applies **reweighting** (redistributes LSTM's 25% to other components)
- Sentiment: 25% → 35%
- Technical: 25% → 35%
- Momentum: 15% → 20%
- Volume: 10% → 10%
- **Confidence reduced** by implicit penalty (~15-20%)

**WHERE IS THIS DOCUMENTED?**
File: `core/paper_trading_coordinator.py` (line 14)
```
* LSTM missing: -20% confidence penalty
* Sentiment missing: -15% confidence penalty
* Volume missing: -10% confidence penalty
```

**ACTUAL IMPLEMENTATION:**
File: `ml_pipeline/swing_signal_generator.py` (lines 217-228)
```python
if not lstm_available:
    # Reweight: Sentiment 35%, Technical 35%, Momentum 20%, Volume 10%
    combined_score = (
        sentiment_score * 0.35 +
        lstm_score * 0.0 +  # LSTM unavailable, use fallback
        technical_score * 0.35 +
        momentum_score * 0.20 +
        volume_score * 0.10
    )
    logger.debug(f"{symbol}: LSTM unavailable, using reweighted components")
```

**CONFIDENCE CALCULATION:**
Without LSTM, the combined_score is lower → confidence is reduced
```python
confidence = min(0.50 + combined_score * 0.5, 0.95)
```

If combined_score drops from 0.6 to 0.48 (due to missing LSTM):
- WITH LSTM: confidence = 0.50 + 0.6 * 0.5 = 0.80 (80%)
- WITHOUT LSTM: confidence = 0.50 + 0.48 * 0.5 = 0.74 (74%)
- **Effective penalty: -6% confidence** (in this example)

OVERNIGHT PIPELINE TRAINING
---------------------------
**File**: `pipelines/models/screening/lstm_trainer.py`

**When**: During overnight AU pipeline execution
**Frequency**: Once per night (typically 22:00-07:00 Sydney time)
**Selection Criteria**: Top 20 stocks by opportunity score

**Configuration** (config/screening_config.json):
```json

  "max_models_per_night": 20,
  "stale_threshold_days": 7,
  "epochs": 50,
  "batch_size": 32,
  "validation_split": 0.2,
  "priority_strategy": "highest_opportunity_score",
  "sequence_length": 60
}
```

**Training Process:**
1. AU overnight pipeline scans 240 ASX stocks
2. Opportunity scorer ranks all stocks by composite score (0-100)
3. LSTM trainer selects **TOP 20** stocks by score
4. For each selected stock:
   - Fetch 2 years of historical data (need 502+ days)
   - Check if existing model is stale (>7 days old)
   - Train new LSTM model (50 epochs, 60-day sequences)
   - Save model to: `finbert_v4.4.4/models/saved_models/{SYMBOL}_lstm_model.h5`
   - Save scaler to: `finbert_v4.4.4/models/saved_models/{SYMBOL}_lstm_scaler.pkl`
   - Update registry: `finbert_v4.4.4/models/saved_models/lstm_models_registry.json`

WHY CBA.AX SHOWS "NOT ENOUGH DATA"
================================================================================

DIAGNOSIS
---------
**Error Message**: "LSTM failed for CBA.AX - not enough data"

**Root Cause**: This error message is **MISLEADING**. The actual issue is:

1. **NO PRE-TRAINED MODEL EXISTS** for CBA.AX
   - Check: `finbert_v4.4.4/models/saved_models/CBA.AX_lstm_model.h5` does not exist
   - Check: CBA.AX not in `lstm_models_registry.json`

2. **WHY NO MODEL?**
   - CBA.AX **was not in the top 20** stocks during last overnight pipeline run
   - Opportunity score for CBA.AX was below the cutoff (e.g., 65.0 vs top stocks 75.0+)
   - Only 20 models are trained per night to keep runtime reasonable (15-25 min total)

3. **FALLBACK BEHAVIOR**
   When dashboard tries to use CBA.AX:
   - SwingSignalGenerator calls `_load_lstm_model('CBA.AX')`
   - Model not found → returns False
   - System tries to train on-the-fly: `_train_lstm_model('CBA.AX', price_data)`
   - **Training fails** if price_data < 200 days (line 648)
   - Error logged: "Insufficient data to train LSTM for CBA.AX"
   - System falls back to **reweighted components** (no LSTM)

**CODE FLOW:**
```
Dashboard loads CBA.AX
  ↓
SwingSignalGenerator.generate_signal('CBA.AX')
  ↓
_analyze_lstm_v185('CBA.AX')
  ↓
_load_lstm_model('CBA.AX')  → Returns False (no pre-trained model)
  ↓
_train_lstm_model('CBA.AX', price_data)
  ↓
if len(price_data) < 200:
    logger.warning("Insufficient data to train LSTM")
    return  ← **ERROR HAPPENS HERE**
  ↓
Use reweighted components (LSTM weight = 0%)
  ↓
Signal generated with reduced confidence (-20%)
```

**WHY "NOT ENOUGH DATA" IS CONFUSING:**
- CBA.AX has **plenty of historical data** (trading since 1991)
- The error refers to the **data passed to the training function**, not market data availability
- If dashboard only loaded 60 days of data (for signal generation), it's insufficient for LSTM training (needs 200+ days)

================================================================================
UNDERSTANDING THE BROADER ISSUE
================================================================================

WHY SELECTIVE LSTM TRAINING?
----------------------------
**Design Decision**: Train only top 20 stocks per night

**Reasons:**
1. **Performance**: Training 240 models takes 4-5 hours (240 × ~1-2 min/model)
2. **Relevance**: Lower-opportunity stocks unlikely to be traded
3. **Resource Management**: Models are ~500 KB each (240 = 120 MB total)
4. **Staleness**: Models >7 days old are retrained automatically
5. **Overnight Window**: Pipeline should complete before market open (10:00 AM AEST)

**Trade-offs:**
- **Pros**: Fast overnight pipeline (15-25 min), focuses on best opportunities
- **Cons**: Lower-ranked stocks like CBA.AX might not have models

**Alternative Approaches:**
1. **Expand to top 50**: Balance between coverage and speed (90 min runtime)
2. **Market-cap weighting**: Always train ASX 50 largest stocks regardless of score
3. **User priority list**: Manually specify "always train" stocks (CBA.AX, BHP.AX, etc.)
4. **Lazy training**: Train on-demand when stock first appears in dashboard (slower initial load)


IS CBA.AX REALLY "LOW OPPORTUNITY"?
-----------------------------------
**Possible Reasons CBA.AX Scored Below Top 20:**

1. **Recent Underperformance**
   - CBA.AX down or sideways in last 1-2 weeks
   - Opportunity scorer penalizes weak momentum
   - Other stocks (mining, energy) scored higher due to commodity boom

2. **Low Volatility**
   - CBA.AX is a stable blue-chip (low volatility)
   - Opportunity scorer favors stocks with 10-20% move potential
   - Banks typically lower opportunity score than miners/energy

3. **Sector Rotation**
   - Financials sector out of favor
   - Materials/Energy sectors dominating top 20
   - Example: BHP.AX, RIO.AX, FMG.AX, WDS.AX all mining stocks in top 10

4. **Technical Factors**
   - CBA.AX RSI neutral (40-60 range)
   - Top 20 stocks have RSI extremes (<30 oversold or >70 breakout)
   - No recent breakouts or momentum signals

**Bottom Line**: CBA.AX is still a **viable trade**, just not in the current top 20 by opportunity score. The LSTM penalty (-20% confidence) is a design limitation, not a reflection of CBA.AX quality as a stock.
