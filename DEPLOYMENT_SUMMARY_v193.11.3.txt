WHAT WAS FIXED (v193.11.3)
================================================================================

ISSUE: Redundant yfinance API Calls Causing UK Pipeline Failures
- TypeError: 'NoneType' object is not subscriptable
- yfinance download failures for UK symbols during market hours
- Problem appeared after macro risk gatekeeper integration (v193.10)

ROOT CAUSE:
- should_allow_trade() was making duplicate yfinance API calls
- enter_position() was calling should_allow_trade() TWICE
- Result: 3 yfinance calls per symbol (each fetching 3mo history)

THE FIX:
✅ Modified should_allow_trade() to accept optional price_data parameter
✅ Updated entry timing evaluation to use passed data (avoid redundant call)
✅ Updated enter_position() to pass price_data to should_allow_trade()
✅ Removed redundant should_allow_trade() call in position sizing

PERFORMANCE IMPROVEMENT:
- Before: 3 yfinance API calls per symbol
- After: 1 yfinance API call per symbol
- Reduction: 67% fewer API calls
