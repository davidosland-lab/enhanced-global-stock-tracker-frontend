FIX #1: ZERO-TRADES THRESHOLD ISSUE (v193.11.6.20)
---------------------------------------------------
Problem: AU pipeline ran 42+ min with AORD +1.6%, generated 0 trades
Root Cause: Three overly conservative filter thresholds
Solution:
  - opportunity_threshold: 65 -> 45 (31% reduction)
  - min_confidence_score: 60 -> 45 (25% reduction)  
  - confidence_threshold: 45 -> 40 (11% reduction)
Expected Impact: 0-2 trades per scan -> 5-12 trades per scan
Status: ✅ COMPLETE (committed in previous session)


FIX #2: GAP PREDICTION ACCURACY (v193.11.6.21)
----------------------------------------------
Problem: AU gap prediction +0.38% vs actual SPI 200 +1.3% (0.92% miss, 242% error)
Root Cause: 
  - Commodities weight too high (10%) relative to predictive power
  - No feedback loop to validate predictions against actual opens
  - Stock scores fixed at prediction time, not adjusted for reality

Solution Components:
1. Weight Adjustment (spi_proxy_realtime.py):
   - US markets: 65% -> 70% (+5%)
   - Commodities: 10% -> 5% (-50%)

2. Gap Reality Checker (NEW: core/gap_reality_checker.py):
   - Stores predictions before market open
   - Validates actual gap 10-15 min after open
   - Calculates accuracy rating (EXCELLENT/GOOD/POOR/TERRIBLE/CATASTROPHIC)
   - Applies score multipliers: 0.80x to 1.20x based on prediction error

3. Integration (paper_trading_coordinator.py):
   - Initialize gap_reality_checker in __init__
   - Store predictions in analyze_pre_market_gaps()
   - New validate_gap_predictions() method
   - Apply multipliers in get_trading_opportunities()

Expected Impact:
  - Prediction accuracy: 242% error -> ~50-80% error (3x improvement)
  - Automatic score adjustment when predictions miss
  - Example: Today's +0.92% miss -> 1.15x multiplier -> stocks boosted 15%

Status: ✅ COMPLETE (committed in this session - 42f7582)


FIX #3: DYNAMIC STOP-LOSS (v193.11.6.21)
----------------------------------------
Problem: Stop-loss not updating when UI slider changed
Root Cause: Static stop-loss calculated at entry, never recalculated
Solution: Added dynamic stop-loss calculation in should_exit_position()
  - Uses tighter of: entry stop OR current UI setting
  - Allows tightening stop-loss after entry
  - Prevents loosening stop-loss (safer)
Status: ✅ COMPLETE (included in COMPREHENSIVE_ISSUE_RESOLUTION_v193.11.6.21.txt)
