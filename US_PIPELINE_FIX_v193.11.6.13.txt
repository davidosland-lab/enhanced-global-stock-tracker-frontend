The dual_regime_analyzer expected EventRiskGuard to have a public method

ISSUE: EventRiskGuard class had private methods:
  - _get_regime_crash_risk() → Returns tuple (regime_label, crash_risk)

ARCHITECTURAL MISMATCH:
-----------------------
dual_regime_analyzer.py (caller) expects public API:
  → get_latest_regime() method

event_risk_guard.py (provider) only provided private API:
  → _get_regime_crash_risk()
  → _get_full_regime_data()

Result: AttributeError when dual_regime_analyzer tries to call missing method.

✅ SOLUTION IMPLEMENTED
================================================================================

APPROACH:
---------
Add a public `get_latest_regime()` method to EventRiskGuard that:
1. Wraps the existing _get_full_regime_data() private method
2. Returns properly formatted regime data dictionary
3. Provides sensible fallback when regime engine unavailable
4. Adds metadata (method='multi_factor', available=True/False)

CODE ADDED:
-----------
Location: pipelines/models/screening/event_risk_guard.py
Lines: 565-589 (25 lines added)

```python
def get_latest_regime(self) -> Dict:
    """
    Public method to get the latest market regime data.
    Used by dual_regime_analyzer.py for comprehensive regime analysis.
    
    Returns:
        Dictionary containing regime_label, crash_risk_score, and other regime data
    """
    regime_data = self._get_full_regime_data()
    
    if regime_data is None:
        # Return default data when regime engine is not available
        return {
            'regime_label': 'UNKNOWN',
            'crash_risk_score': 0.0,
            'method': 'multi_factor',
            'confidence': 'LOW',
            'available': False
        }
    
    # Add method identifier
    regime_data['method'] = 'multi_factor'
    regime_data['available'] = True
    
    return regime_data
```

KEY FEATURES:
-------------
✅ Public method - accessible from dual_regime_analyzer
✅ Returns Dict with standard fields (regime_label, crash_risk_score, etc.)
✅ Graceful fallback when regime engine unavailable
✅ Adds 'method': 'multi_factor' to identify data source
✅ Adds 'available': True/False to indicate data validity
