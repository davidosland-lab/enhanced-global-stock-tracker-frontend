ISSUE 1: STOP-LOSS BUG - FIXED
================================================================================

PROBLEM:
User set stop-loss to 2% but position dropped 3% and did not sell.

ROOT CAUSE:
position.stop_loss was STATIC (set at entry time). When user changed UI
setting from 4% to 2% AFTER opening position, the old 4% stop-loss remained.

Example:
- Position opened with 4% stop: entry USD100.00, stop USD96.00
- User changed UI to 2% (should be: USD98.00)
- Price dropped to USD97.00 (3% loss)
- Expected: SELL (3% > 2%)
- Actual: HOLD (USD97.00 > USD96.00 old stop)

THE FIX (v193.11.6.21):
File: core/paper_trading_coordinator.py
Line: ~2244-2247 -> 2244-2266 (expanded)

NEW CODE:
```python
# FIX v193.11.6.21: Dynamic stop-loss respects current UI setting
stop_loss_pct = abs(self.ui_default_stop_loss) if self.ui_default_stop_loss is not None else self.config['swing_trading']['stop_loss_percent']
dynamic_stop_loss = position.entry_price * (1 - stop_loss_pct / 100)

# Use the TIGHTER of: entry stop_loss OR current UI setting
effective_stop_loss = max(position.stop_loss, dynamic_stop_loss)

if price <= effective_stop_loss:
    loss_pct = ((price - position.entry_price) / position.entry_price) * 100
    
    if effective_stop_loss == dynamic_stop_loss:
        logger.info(f"[STOP-LOSS] {symbol}: Dynamic stop triggered (UI: {stop_loss_pct}%)")
        return f"STOP_LOSS_DYNAMIC_{abs(int(loss_pct))}%"
    else:
        logger.info(f"[STOP-LOSS] {symbol}: Entry stop triggered")
        return f"STOP_LOSS_STATIC_{abs(int(loss_pct))}%"
```

BEHAVIOR CHANGE:
- Tightening stop-loss after entry NOW WORKS (e.g., 4% -> 2%)
- Loosening stop-loss after entry uses entry stop (safer)
- User's current setting is always respected

ISSUE 3: OPENING GAP IMPACT ON STOCK SCORES
================================================================================

QUESTION:
"Is the market opening this high a measure of sentiment and could it impact
the stock scores?"

SHORT ANSWER: YES, IT SHOULD.

CURRENT BEHAVIOR:
1. Pipeline runs BEFORE market open (e.g., 8:00 AM)
2. Predicts gap: +0.38% (in this case)
3. Scans 240 stocks, scores them (0-100)
4. Market ACTUALLY opens +1.3% (92bp higher than predicted)
5. Stock scores DO NOT adjust for actual gap vs predicted gap

PROBLEM:
If market opens much STRONGER than predicted, stocks should get:
- Higher momentum scores (market enthusiasm)
- Higher sentiment adjustments (risk-on environment)
- Potential position size increases (strong open = strong day)

If market opens much WEAKER than predicted, stocks should get:
- Lower scores (risk-off)
- Tighter stop-losses
- Smaller position sizes or delayed entries

PROPOSED SOLUTION: POST-OPEN GAP ADJUSTMENT MODULE
---------------------------------------------------

NEW MODULE: pipelines/models/screening/gap_reality_adjuster.py

Purpose:
- Compare predicted gap vs actual opening gap
- Adjust stock scores based on gap surprise
- Trigger mid-morning rescan if gap miss >0.5%

Logic:
```python
def adjust_scores_for_gap_reality(self, stocks, predicted_gap, actual_gap):
    gap_surprise = actual_gap - predicted_gap  # +0.92% in your case
    
    if abs(gap_surprise) < 0.3:
        # Small miss, no adjustment
        return stocks
    
    # Large miss - adjust scores
    adjustment_factor = gap_surprise / predicted_gap if predicted_gap != 0 else 0
    
    for stock in stocks:
        original_score = stock['opportunity_score']
        
        # Positive surprise: boost scores (market stronger than expected)
        if gap_surprise > 0.5:
            adjusted_score = original_score * (1 + adjustment_factor * 0.15)
            stock['opportunity_score'] = min(100, adjusted_score)
            stock['gap_adjusted'] = True
            stock['gap_surprise'] = gap_surprise
        
        # Negative surprise: reduce scores (market weaker than expected)
        elif gap_surprise < -0.5:
            adjusted_score = original_score * (1 + adjustment_factor * 0.15)
            stock['opportunity_score'] = max(0, adjusted_score)
            stock['gap_adjusted'] = True
            stock['gap_surprise'] = gap_surprise
    
    return stocks
```

Method: run() or get_trading_opportunities()
