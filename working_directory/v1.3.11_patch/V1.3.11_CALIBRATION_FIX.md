## 🚨 Critical Issue Identified

**Problem:** Market performance charts were showing percentage changes calculated from the **market open price** instead of the **previous day's close**.

### User Reported Discrepancies:
```
Index       | Actual Close % | Dashboard Showed | Issue
------------|----------------|------------------|-------
NASDAQ      | -0.03%         | Wrong            | ❌
S&P 500     | +0.66%         | Wrong            | ❌
FTSE 100    | +0.2%          | Wrong            | ❌
```

### Root Cause:
The code calculated percentage change from `first_price` (market open), not from previous day's close:
```python
# WRONG (v1.3.10)
first_price = market_hours_data['Close'].iloc[0]  # Market open
pct_change = ((row['Close'] - first_price) / first_price) * 100
```

---

## ✅ Solution Implemented (v1.3.11)

### What Changed:
1. **Fetch 2 Days of History** - ensures we have previous close available
2. **Calculate Previous Close** - finds last close before current trading day
3. **Use Previous Close as Reference** - percentage change now matches official figures

### New Code Logic:
```python
# CORRECT (v1.3.11)
# Get previous day's close (reference price for % change)
previous_day_data = hist[hist.index.date < latest_date]

if len(previous_day_data) > 0:
    # Use previous day's last close as reference
    previous_close = previous_day_data['Close'].iloc[-1]
else:
    # Fallback: use first price of current session if no previous data
    previous_close = market_hours_data['Close'].iloc[0]

# Calculate percentage change from previous close
pct_change = ((row['Close'] - previous_close) / previous_close) * 100
```
