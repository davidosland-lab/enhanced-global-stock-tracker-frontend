🔴 CRITICAL BUGS FIXED
──────────────────────────────────────────────────────────────────────────

### BUG 1: FinBERT Reinitialization Every 5 Seconds

**SYMPTOM:**
- Dashboard logs showed repeated "Initializing FinBERT v4.4.4..." messages
- Occurred every 5 seconds (dashboard refresh interval)
- Performance degradation and memory overhead

**ROOT CAUSE:**
- Dashboard created new `IntegratedSentimentAnalyzer()` instance every update
- Line 1608: `sentiment_int = IntegratedSentimentAnalyzer()`  ← NEW INSTANCE!
- FinBERT model (400+ MB) reloaded from disk every 5 seconds

**FIX:**
✅ Added global cached sentiment analyzer instance
✅ Implemented thread-safe double-checked locking pattern
✅ `get_sentiment_analyzer()` helper returns cached instance
✅ FinBERT initialized ONCE at startup, then reused

### BUG 2: Dashboard Not Showing Live Prices

**SYMPTOM:**
- User report: "Three buy signals triggered, but dashboard showed no change 
  after 2 hours of trading. RIO.AX is one of the stocks."
- Positions frozen at entry prices
- P&L calculations showed 0.0% despite market movement

**ROOT CAUSE:**
- Dashboard displayed cached prices from state object
- Line 1795: `current_price = pos.get('current_price', pos['entry_price'])`
- Line 1565: `unrealized_pnl = state['positions']['unrealized_pnl']`
- State only updated when trading system runs (not every 5 seconds)

**FIX:**
✅ Added live price fetching using yfinance for each position
✅ Recalculates unrealized P&L from live prices every 5 seconds
✅ Displays current market prices in positions list
✅ Graceful fallback to state values if API fails
