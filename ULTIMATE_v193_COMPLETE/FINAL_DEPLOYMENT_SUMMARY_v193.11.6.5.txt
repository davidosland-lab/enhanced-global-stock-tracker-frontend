BUG #1 - Missing Methods (Fixed in v193.11.6.3)
--------------------------------------------------------------------------------
ISSUE: AttributeError: 'SPIMonitor' object has no attribute '_get_us_markets'
       Also: '_get_default_sentiment' was missing

LOCATION: pipelines/models/screening/spi_monitor.py
  - Line 119: self._get_us_markets() 
  - Lines 116, 123, 220: self._get_default_sentiment()

ROOT CAUSE:
  Method calls existed but methods were never implemented

SOLUTION:
  Added two methods (lines 388-423):
  
  def _get_us_markets(self) -> Dict:
      """Alias for backward compatibility"""
      return self._get_us_market_data()
  
  def _get_default_sentiment(self) -> Dict:
      """Return neutral sentiment data as fallback"""
      return {
          'sentiment_score': 50.0,
          'predicted_gap_pct': 0.0,
          'confidence': 0.5,
          'direction': 'NEUTRAL',
          ...
      }

IMPACT: Market sentiment can now handle missing data gracefully


BUG #2 - Wrong Argument Order (Fixed in v193.11.6.4)
--------------------------------------------------------------------------------
ISSUE: AttributeError: 'float' object has no attribute 'values'

LOCATION: pipelines/models/screening/spi_monitor.py
  - Line 174-178: Call to _calculate_sentiment_score()
  - Line 649: us_changes = [data['change_pct'] for data in us_data.values()]

ROOT CAUSE:
  Method signature: _calculate_sentiment_score(us_data: Dict, gap_prediction: Dict, asx_data: Dict)
  
  Incorrect call:
    sentiment_score = self._calculate_sentiment_score(
        gap_prediction['predicted_gap_pct'],  # ❌ float, should be Dict
        gap_prediction.get('confidence', 0.7), # ❌ float, should be Dict
        us_data                                # ❌ wrong position
    )

SOLUTION:
  Corrected call (lines 174-178):
    sentiment_score = self._calculate_sentiment_score(
        us_data,        # ✅ Dict in position 1
        gap_prediction, # ✅ Dict in position 2
        asx_data        # ✅ Dict in position 3
    )

IMPACT: Sentiment score calculation now works correctly

BUG #4 - Logger Initialization Order (Fixed in v193.11.6.5)
--------------------------------------------------------------------------------
ISSUE: NameError: name 'logger' is not defined

LOCATION: pipelines/models/screening/spi_monitor.py
  - Import section: Logger was used before being created

ROOT CAUSE:
  Logger was referenced in conditional import block before being initialized

SOLUTION:
  Moved logger initialization to top of file (before any use):
    # Set up logging first
    logging.basicConfig(...)
    logger = logging.getLogger(__name__)
    
    # Then do imports that might use logger
    try:
        from .spi_proxy_advanced import SPIProxy
        ...

IMPACT: Module can now be imported without errors

🔧 TROUBLESHOOTING
================================================================================

ISSUE: "Module not found" errors during installation
--------------------------------------------------------------------------------
SOLUTION:
  1. Close all Python/terminal windows
  2. Re-run INSTALL_FIRST_TIME.bat
  3. Wait for full completion (10-15 minutes)
  4. Re-run INSTALL_COMPLETE_v193.bat

ISSUE: Web Control Center won't start (port 5000 already in use)
--------------------------------------------------------------------------------
SOLUTION:
  1. Open Task Manager
  2. End any "python.exe" processes
  3. Try again
  
  OR
  
  netstat -ano | findstr :5000
  taskkill /PID <PID> /F

ISSUE: Market sentiment still showing errors
--------------------------------------------------------------------------------
VERIFICATION:
  1. Check VERSION.json shows 1.3.15.193.11.6.5
  2. Check spi_monitor.py file date is 2026-03-08
  3. Verify installation:
     python -c "from pipelines.models.screening.spi_monitor import SPIMonitor; m = SPIMonitor(); print('OK')"

SOLUTION:
  If verification fails, re-extract ZIP (may have partial upgrade)

ISSUE: Pipelines start but show no data
--------------------------------------------------------------------------------
CAUSES:
  - No internet connection
  - Market closed (expected for some pipelines)
  - Yahoo Finance API temporary unavailability

SOLUTION:
  - Check internet connection
  - Try --ignore-market-hours flag
  - Wait a few minutes and retry
