CRITICAL BUG FIXED
==================
Issue: Positions randomly changed quantities or disappeared without proper trade 
records, causing:
- Inconsistent position counts (e.g., NAB.AX 277 shares → disappeared)
- Share quantities changing (JUP.L 85→98, CNA.L 65→75)
- Total trades stuck at 0
- Capital accounting mismatches
- State file corruption

ROOT CAUSE ANALYSIS
===================
Location: paper_trading_coordinator.py, line ~2130
Problem: enter_position() overwrites existing positions without proper exit

Code Flow:
1. Pipeline recommendations call enter_position() for symbol X
2. Regular scan also calls enter_position() for same symbol X
3. Line 2130: self.positions[symbol] = position  ← OVERWRITES!
4. Old position object is lost (no exit_position() called)
5. No trade record created, capital not updated
6. State file saves incomplete data

Example from logs:
Time 16:16:08: NAB.AX 277 shares @ $47.35 (position created)
Time 16:16:44: NAB.AX missing (overwritten by new entry attempt)
Time 16:17:22: Different positions appear (continuous overwrites)

THE FIX
=======
Added duplicate entry guard in enter_position():

```python
def enter_position(self, symbol: str, signal: Dict) -> bool:
    """Enter a new swing position (paper trade)"""
    
    # v193.11.7.3: Prevent position overwrite bug
    if symbol in self.positions:
        logger.warning(
            f"[BLOCKED] {symbol}: Cannot enter - position already exists. "
            f"Current: {self.positions[symbol].quantity} shares @ "
            f"${self.positions[symbol].entry_price:.2f}"
        )
        return False
    
    # ... rest of entry logic
```

This ensures:
✓ Only one position per symbol at a time
✓ Must call exit_position() before re-entering
✓ Proper trade records and capital accounting
✓ No state file corruption
