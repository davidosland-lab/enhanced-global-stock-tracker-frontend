🐛 PROBLEM DESCRIPTION
───────────────────────────────────────────────────────────────

ISSUE: Yahoo Finance API Rate Limiting

SYMPTOMS:
- Dashboard refreshing every 5 seconds
- Yahoo Finance returning HTTP 401 errors
- "Invalid Crumb" errors
- "Too Many Requests. Rate limited" errors
- "User is unable to access this feature" errors
- Live prices not updating
- Market charts showing stale data
- Paper trading positions not updating

LOG EVIDENCE:
```
yfinance - ERROR - HTTP Error 401: Invalid Crumb
yfinance - ERROR - HTTP Error 401: Unauthorized
__main__ - WARNING - Too Many Requests. Rate limited
```

ROOT CAUSE:
- Dashboard interval set to 5 seconds (too frequent)
- Yahoo Finance limits: ~2000 requests/hour per IP
- At 5-second intervals: 720 requests/hour per symbol
- With 4+ symbols: >2880 requests/hour (OVER LIMIT)

IMPACT:
- ❌ Dashboard appears broken (no live updates)
- ❌ Charts not reflecting market movements  
- ❌ Paper trading can't update positions
- ⚠️ FinBERT reinitialized repeatedly (performance issue)

🔧 FIX APPLIED
───────────────────────────────────────────────────────────────

FILE: core/unified_trading_dashboard.py
LINE: 1061

BEFORE:
```python
dcc.Interval(
    id='interval-component',
    interval=5*1000,  # Update every 5 seconds
    n_intervals=0
),
```

AFTER:
```python
dcc.Interval(
    id='interval-component',
    interval=30*1000,  # Update every 30 seconds (v193.3 - reduced to avoid Yahoo rate limiting)
    n_intervals=0
),
```

CHANGE:
- Refresh interval: 5 seconds → 30 seconds (6x reduction)
- Request rate: 720/hour → 120/hour per symbol
- Total requests (4 symbols): 2880/hour → 480/hour
- Result: Well under Yahoo's 2000/hour limit
