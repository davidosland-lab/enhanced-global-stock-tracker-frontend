====================================================================================
PROBLEM STATEMENT
====================================================================================

Windows users experienced UnicodeEncodeError when running the trading pipelines:

  UnicodeEncodeError: 'charmap' codec can't encode character '\u2713' in 
  position 52: character maps to <undefined>

This error occurred in logging/__init__.py when logger calls contained Unicode 
special characters (checkmarks, arrows, emojis) that cannot be encoded in the 
Windows cp1252 console encoding.

The error was triggered by messages like:
  logger.info(f"✓ Fetched {len(articles)} articles for {symbol}")
  logger.info(f"[SPI PROXY] ✓ Success! Gap: {result['proxy_move']:+.2f}%")
  logger.error(f"❌ LSTM MODEL NOT TRAINED for {symbol}")

====================================================================================
ROOT CAUSE
====================================================================================

1. Logger calls in 20+ files contained Unicode special characters
2. Windows console defaults to cp1252 encoding (not UTF-8)
3. Despite v193.11.1 fixing UTF-8 in logging handlers, some Unicode chars 
   (like ✓ U+2713, ❌ U+274C) are not available in cp1252
4. Python's logging module fails when trying to write these characters to stdout

====================================================================================
SOLUTION
====================================================================================

Created automated hotfix script that replaced ALL Unicode characters in logger 
calls with ASCII equivalents:

  ✓ (U+2713) → [OK]
  ✗ (U+2717) → [X]
  → (U+2192) → ->
  ⚠ (U+26A0) → [!]
  🚨 (U+1F6A8) → [ALERT]
  ❌ (U+274C) → [ERROR]
  🔒 (U+1F512) → [LOCKED]
  📦 (U+1F4E6) → [CACHE]

PREVENTION
====================================================================================

Going forward, ALL new code must follow these rules:

1. NEVER use Unicode special characters in logger calls
2. Use ASCII equivalents: [OK], [X], [!], [ERROR], [ALERT]
3. Use -> instead of → for arrows
4. Avoid emojis entirely in production logging
5. If Unicode is required (e.g., user-facing output), use .encode('ascii', 'replace')

Example:
  ❌ logger.info(f"✓ Success for {symbol}")
  ✅ logger.info(f"[OK] Success for {symbol}")
