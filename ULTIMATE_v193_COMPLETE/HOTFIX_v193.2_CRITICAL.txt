🐛 BUG DESCRIPTION
───────────────────────────────────────────────────────────────

ROOT CAUSE: Variable Scope Error
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Location: All 3 pipelines (AU, UK, US)
- pipelines/models/screening/overnight_pipeline.py
- pipelines/models/screening/uk_overnight_pipeline.py
- pipelines/models/screening/us_overnight_pipeline.py

ISSUE: The variable `macro_news` is only defined INSIDE the 
Phase 1.3 try block. When Phase 1.4 (World Event Risk) tries 
to access it, the variable doesn't exist in these scenarios:

1. ❌ If macro_monitor is None (Phase 1.3 skipped entirely)
2. ❌ If an exception occurs in Phase 1.3
3. ❌ Even when Phase 1.3 succeeds, exception handler doesn't 
      set the local variable

Result: Phase 1.4 crashes or always returns neutral (50/100)
