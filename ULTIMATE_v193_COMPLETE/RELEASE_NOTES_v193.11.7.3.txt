FIX #2: SIGNAL GENERATION CRASH (v193.11.7.1)
================================================================================

ISSUE:
------
Error: "unsupported format string passed to NoneType.__format__"
ALL symbols failing signal generation
ZERO trades executing (Total Trades: 0)
$100,000 cash sitting idle, no positions opened

ROOT CAUSE:
-----------
Component scores (sentiment/LSTM/technical) could be None

Safe extraction of component values before formatting:

components = enhanced_signal.get('components', {})

lstm_val = components.get('lstm', 0) if components.get('lstm') is not None else 0
technical_val = components.get('technical', 0) if components.get('technical') is not None else 0

logger.info(f"Components: Sentiment={sentiment_val:.3f}, LSTM={lstm_val:.3f}, Technical={technical_val:.3f}")

RESULT:
-------
✅ Signal generation succeeds for all symbols
✅ Log entries: "[OK] AAPL Signal: BUY (conf=65.23) | Components: ..."
✅ Trades execute when confidence > threshold
✅ No more "unsupported format string" errors

FIX #3: MARKET HOURS ENFORCEMENT (v193.11.7.2)
================================================================================

ISSUE:
------
User reported: "NAB.AX purchased after market has closed"
System logged: "NAB.AX (After-hours)" but still executed trades
Losses accumulating on closed markets
All symbols scanned regardless of market status

ROOT CAUSE:
-----------
Market hours check logged warnings but didn't prevent trading
No actual filtering before trading loop
No guard in enter_position() function

FIX:
----
Three-layer protection:

1. Filter symbols by market status:
   open_symbols = []
   for symbol in self.symbols:
       can_trade, reason = market_calendar.can_trade_symbol(symbol)
       if can_trade:
           open_symbols.append(symbol)

2. Scan only open markets:
   for symbol in open_symbols:  # Only check tradeable markets
       should_enter, confidence, signal = self.evaluate_entry(symbol)

3. Safety check in enter_position():
   if not signal.get('type') == 'MANUAL' and MARKET_CALENDAR_AVAILABLE:
       can_trade, reason = market_calendar.can_trade_symbol(symbol)
       if not can_trade:
           logger.warning(f"{symbol}: TRADE BLOCKED - Market closed")
           return False

RESULT:
-------
✅ NAB.AX (After-hours) → NO TRADES (correctly skipped)
✅ JUP.L (Pre-market) → NO TRADES (correctly skipped)
✅ BAC (Market closed) → NO TRADES (correctly skipped)
✅ Log: "[CALENDAR] Trading only on open markets: X of Y symbols"
✅ Manual trades can still override (for testing)

FIX #4: POSITION OVERWRITE PREVENTION (v193.11.7.3)
================================================================================

ISSUE:
------
User reported: "Positions appearing/disappearing randomly"
- NAB.AX: 277 shares → disappeared → 225 shares (different quantity!)
- JUP.L: 85 shares → 98 shares (increased without trade)
- CNA.L: 65 shares → 75 shares (increased without trade)
- Total Trades: 0 (no trades being recorded)
- No "[OK] POSITION CLOSED" messages
- Dashboard showing conflicting states (traded/not traded)

ROOT CAUSE:
-----------
Position Overwrite Bug in enter_position():
Line 2130: self.positions[symbol] = position

If enter_position() called twice for same symbol:
• Overwrites old position with new one
• Old position destroyed without proper exit
• No trade record created (Total Trades stays 0)
• Capital accounting breaks
• Position shares change mysteriously

Multiple entry points can cause duplicates:
- Regular scan loop
- Pipeline recommendations
- Manual trades
- Pre-market gap trading
- Dashboard operations (race conditions)

FIX:
----
Guard check at start of enter_position():

if symbol in self.positions:
    existing = self.positions[symbol]
    logger.warning(f"{symbol}: TRADE BLOCKED - Position already exists")
    logger.warning(f"  Existing: {existing.shares} shares @ USD{existing.entry_price:.2f}")
    logger.warning(f"  Current P&L: {existing.unrealized_pnl_pct:+.2f}%")
    logger.warning(f"  Attempted: New entry would OVERWRITE existing position!")
    logger.warning(f"  → Use exit_position() first to close current position")
    return False

RESULT:
-------
✅ Positions cannot be overwritten
✅ Duplicate entry attempts blocked and logged
✅ Trade counts only increment on proper exit
✅ Capital accounting stays consistent
✅ Position share counts never change without trade
✅ Dashboard shows consistent state
✅ No more positions disappearing

SUPPORT & TROUBLESHOOTING
================================================================================

If Signal Generation Still Fails:
----------------------------------
1. Check logs/paper_trading.log for detailed errors
2. Verify Python 3.12+ installed: python --version
3. Reinstall dependencies: pip install -r requirements.txt
4. Check ml_pipeline modules accessible

If After-Hours Trading Still Occurs:
-------------------------------------
1. Look for "TRADE BLOCKED - Market closed" in logs
2. Verify MARKET_CALENDAR_AVAILABLE is True
3. Check symbol suffixes (.AX, .L, etc.)
4. Review market_calendar configuration

If Positions Still Disappear:
------------------------------
1. Look for "TRADE BLOCKED - Position already exists" messages
2. Count how many times this occurs (indicates source of duplicates)
3. Check state/paper_trading_state.json for consistency
4. Verify Total Trades increments only with "[OK] POSITION CLOSED"

If Trading Loop Still Crashes:
-------------------------------
1. Check logs for exception details
2. Verify exception is logged but loop continues
3. Look for "[WARN] Trading cycle failed but loop will continue"
4. Check system resources (RAM, disk space)
