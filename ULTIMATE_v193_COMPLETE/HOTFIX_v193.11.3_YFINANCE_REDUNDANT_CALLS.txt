HOTFIX v193.11.3 - YFINANCE REDUNDANT API CALLS FIX

================================================================================
ISSUE SUMMARY
================================================================================

The UK pipeline was experiencing yfinance download failures during London Stock 
Exchange trading hours (08:00-16:30 GMT), with errors like:

  TypeError: 'NoneType' object is not subscriptable
  
This occurred for many UK symbols (HSBA.L, LLOY.L, SHEL.L, GSK.L, etc.) even 
though the market was OPEN at the time (09:40 GMT).

User correctly identified: "This only occurred after the changes to buying 
logic were implemented."

================================================================================
ROOT CAUSE
================================================================================

After integrating the Macro Risk Gatekeeper (v193.10) and entry timing logic
(v1.3.15.163), the code was making REDUNDANT yfinance API calls:

CALL #1: evaluate_entry() fetches 3mo history at line 1556
  ↓
CALL #2: should_allow_trade() fetches 3mo history AGAIN at lines 1169-1171
  ↓
CALL #3: should_allow_trade() called AGAIN for position sizing at line 1726

Result: 3 API calls per symbol, causing:
- Rate-limiting
- Timing window issues
- Network errors
- TypeError when API returns None

================================================================================
THE FIX
================================================================================

1. Modified should_allow_trade() signature to accept optional price_data:
   
   Before:
   def should_allow_trade(self, symbol: str, signal: Dict, sentiment_score: float)
   
   After:
   def should_allow_trade(self, symbol: str, signal: Dict, sentiment_score: float, price_data=None)

2. Updated entry timing evaluation to use passed data:
   
   Before (lines 1169-1171):
   import yfinance as yf
   ticker = yf.Ticker(symbol)
   hist = ticker.history(period='3mo')
   
   After:
   if price_data is None or price_data.empty:
       import yfinance as yf
       ticker = yf.Ticker(symbol)
       hist = ticker.history(period='3mo')
   else:
       hist = price_data  # Use pre-fetched data!

3. Updated enter_position() to pass price_data (line 1692):
   
   Before:
   gate, position_multiplier, reason = self.should_allow_trade(symbol, signal, self.last_market_sentiment)
   
   After:
   price_data = self.fetch_market_data(symbol, period="3mo")
   gate, position_multiplier, reason = self.should_allow_trade(symbol, signal, self.last_market_sentiment, price_data)

4. Removed redundant call in position sizing (line 1726):
   
   Before:
   gate, position_multiplier, reason = self.should_allow_trade(symbol, signal, self.last_market_sentiment)
   position_size = base_size * position_multiplier
   
   After:
   # Use position_multiplier already calculated above
   position_size = base_size * position_multiplier
