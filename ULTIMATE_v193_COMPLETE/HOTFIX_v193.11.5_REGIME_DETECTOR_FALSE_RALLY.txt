================================================================================
ISSUE DESCRIPTION
================================================================================

CRITICAL BUG: Market Regime Detector was defaulting to "US_BROAD_RALLY" when
unable to determine market regime, even during down markets.

SYMPTOMS:
---------
User reported AU pipeline log showing:
  "US Markets: S&P -0.6%, NASDAQ -0.3%"
  "Volatility: VIX 23.8"
  "Regime detected: MarketRegime.US_BROAD_RALLY (strength: 0.30, confidence: 0.45)"
  "Explanation: Broad US market rally. ASX may see modest gains..."

This is WRONG - when S&P and NASDAQ are both negative and VIX is elevated,
the market is clearly in risk-off mode, not rallying.

IMPACT:
-------
❌ False bullish signals during down markets
❌ Incorrect sector allocations (favoring cyclicals when should be defensive)
❌ Aggressive position sizing when should be cautious
❌ Risk management gates may not trigger appropriately
❌ Misleading logs confuse operators
❌ Potential trading losses from bullish bias in bearish conditions

AFFECTED COMPONENTS:
--------------------
- models/market_regime_detector.py (line 360)
- All AU pipeline runs (uses regime for sector scoring)
- Opportunity scorer (weights recommendations by regime)
- Risk management decisions
- Position sizing calculations

================================================================================
ROOT CAUSE ANALYSIS
================================================================================

FILE: models/market_regime_detector.py
FUNCTION: _detect_primary_regime()
LINE: 360

BUGGY CODE:
-----------
```python
# Select primary regime (highest score)
if scores:
    primary = max(scores, key=scores.get)
    strength = min(scores[primary], 1.0)
    return primary, strength

# Default to neutral  ← COMMENT SAYS "NEUTRAL"
return MarketRegime.US_BROAD_RALLY, 0.3  ← CODE SAYS "RALLY"!!!
```

PROBLEM:
--------
1. Comment says "Default to neutral"
2. Code returns "US_BROAD_RALLY" (bullish)
3. When scores dict is empty or no regime matches, falls through to this default
4. Creates false positive bullish signal in uncertain/down markets

WHY THIS HAPPENED:
------------------
- Likely copy-paste error or incomplete refactoring
- Comment was updated but code wasn't
- No unit tests catching edge case of empty scores dict
- Fallback logic wasn't validated against various market conditions

================================================================================
THE FIX
================================================================================

CHANGED LINE 360 FROM:
----------------------
```python
# Default to neutral
return MarketRegime.US_BROAD_RALLY, 0.3
```

TO:
---
```python
# Default to risk-off when unable to determine regime (conservative approach)
logger.warning("[REGIME] Unable to determine regime from market data, defaulting to US_RISK_OFF")
return MarketRegime.US_RISK_OFF, 0.3
```

RATIONALE:
----------
✅ Conservative approach: better to be cautious than overly aggressive
✅ US_RISK_OFF is a valid regime in the MarketRegime enum
✅ Matches market reality when detection fails (typically during uncertain/volatile times)
✅ Aligns with risk management principles: when in doubt, protect capital
✅ Added warning log for visibility and debugging
✅ Operator sees "[REGIME] Unable to determine regime..." in logs
✅ Maintains same strength value (0.3) indicating low confidence

ALTERNATIVE CONSIDERED:
-----------------------
Could have used COMMODITY_MIXED (the only neutral option in enum) but:
- Less appropriate for equity trading
- Doesn't convey the caution needed when regime is unclear
- US_RISK_OFF better matches "when uncertain, be defensive" philosophy
