CRITICAL BUG FIXED: NoneType.__format__ Error
----------------------------------------------

SYMPTOMS:
---------
• Error message: "Error generating signal for <symbol>: unsupported format string passed to NoneType.__format__"
• ALL symbols failing to generate signals
• No trades executing despite system running
• Capital sitting idle ($100,000 cash, $0 invested)
• Portfolio showing: Total Trades 0, Win Rate 0.0%

ROOT CAUSE:
-----------
In paper_trading_coordinator.py, when logging signal components, the code attempted
to format None values in f-strings:

  BAD CODE (line ~1688):
  ----------------------
  logger.info(
      f"Components: Sentiment={enhanced_signal.get('components', {}).get('sentiment', 0):.3f}, "
      f"LSTM={enhanced_signal.get('components', {}).get('lstm', 0):.3f}, "
      f"Technical={enhanced_signal.get('components', {}).get('technical', 0):.3f}"
  )

The .get() method with a default of 0 doesn't prevent None if the key exists but
has a None value. When Python tries to format None with :.3f, it crashes with:
"unsupported format string passed to NoneType.__format__"

THE FIX:
--------
Extract component values safely and check for None BEFORE formatting:

  FIXED CODE (line ~1688):
  ------------------------
  components = enhanced_signal.get('components', {})
  sentiment_val = components.get('sentiment', 0) if components.get('sentiment') is not None else 0
  lstm_val = components.get('lstm', 0) if components.get('lstm') is not None else 0
  technical_val = components.get('technical', 0) if components.get('technical') is not None else 0
  
  logger.info(
      f"[OK] {symbol} Signal: {enhanced_signal['prediction']} "
      f"(conf={enhanced_signal['confidence']:.2f}) | "
      f"Components: Sentiment={sentiment_val:.3f}, "
      f"LSTM={lstm_val:.3f}, "
      f"Technical={technical_val:.3f}"
  )
