USER-DISCOVERED ANOMALY
================================================================================

User Report:
  "Why is there a Total impact of +0.53 percentage points?"
  
  Log Output:
    Original Gap: -0.75%
    Sentiment Score: 3.4/100 (deviation: -0.93)
    World Risk: 87.6/100
    Risk Multiplier: 0.98x
    Adjusted Gap: -0.22%
    Total Impact: +0.53 percentage points  ← ❌ WRONG DIRECTION!

Analysis:
  When markets are NEGATIVE (-0.75%) and sentiment is EXTREMELY BEARISH (3.4/100)
  and world risk is HIGH (87.6/100), the system should AMPLIFY the negative move,
  NOT reduce it!
  
  Expected: Adjusted Gap should be MORE negative (e.g., -1.0% or -1.5%)
  Actual: Adjusted Gap became LESS negative (-0.22%)
  
  This is BACKWARDS logic - bearish sentiment was making things less bearish!

================================================================================
ROOT CAUSE ANALYSIS
================================================================================

OLD FORMULA (WRONG):
  adjusted_gap = original_gap * (1 + (sentiment_deviation * sentiment_factor))
  
  With user's data:
    sentiment_deviation = (3.4 - 50) / 50 = -0.932
    sentiment_factor = 0.75 (RISK-OFF regime)
    adjustment = -0.932 * 0.75 = -0.699
    adjusted_gap = -0.75 * (1 + (-0.699))
    adjusted_gap = -0.75 * 0.301
    adjusted_gap = -0.226%  ← Less negative than -0.75%!

Problem:
  When sentiment_deviation is negative (bearish sentiment):
  - (1 + negative_value) becomes (1 - positive_value)
  - Multiplying by a value < 1.0 REDUCES the magnitude
  - So negative gaps became LESS negative (wrong!)
  - And positive gaps would become MORE positive (also wrong!)

The formula was designed for bullish sentiment but broke with bearish sentiment.

================================================================================
SOLUTION IMPLEMENTED
================================================================================

NEW LOGIC (CORRECT):

Step 1: Determine if sentiment and gap direction AGREE
  gap_is_negative = original_gap < 0
  sentiment_is_bearish = final_sentiment < 50
  signals_agree = (gap_is_negative AND sentiment_is_bearish) OR
                  (gap_is_positive AND sentiment_is_bullish)

Step 2: Apply appropriate adjustment
  IF signals AGREE:
    → AMPLIFY the move
    adjustment_magnitude = abs(sentiment_deviation) * sentiment_factor
    adjusted_gap = original_gap * (1 + adjustment_magnitude)
    
  IF signals DISAGREE:
    → DAMPEN the move
    adjustment_magnitude = abs(sentiment_deviation) * sentiment_factor * 0.5
    adjusted_gap = original_gap * (1 - adjustment_magnitude)

Step 3: Apply risk multiplier for extreme conditions (World Risk >= 85)
  IF signals AGREE:
    → Further AMPLIFY (1.0x → 1.5x)
    risk_multiplier = 1.0 + (risk_intensity * 0.5)
    
  IF signals DISAGREE:
    → Further DAMPEN (1.0x → 0.6x)
    risk_multiplier = 1.0 - (risk_intensity * 0.4)
