================================================================================
  EXECUTIVE SUMMARY
================================================================================

**Problem**: Dashboard showed stale prices for UK stocks (BP.L) during
after-hours/overnight, while other stocks (RIO.AX, LGEN.L) updated correctly.

**Root Cause**: The `fetch_current_price()` method only fetched `regularMarketPrice`,
which is only available during market hours. When markets close, it returns None,
causing the dashboard to display the entry price indefinitely.

**Solution**: Enhanced the method with a 4-tier fallback system:
1. regularMarketPrice (live trading)
2. postMarketPrice (after-hours)
3. preMarketPrice (before open)
4. regularMarketPreviousClose (market closed)
5. yfinance fallback

**Status**: Fix applied to v1.3.15.191, ready for testing

**Issues**:

**AFTER (v191)** - FIXED CODE:
```python
def fetch_current_price(self, symbol: str) -> Optional[float]:
    if YAHOOQUERY_AVAILABLE:
        ticker = Ticker(symbol)
        quote = ticker.price
        
        if isinstance(quote, dict) and symbol in quote:
            stock_data = quote[symbol]
            
            # 1. Try regular market price (live)
            price = stock_data.get('regularMarketPrice')
            if price and price > 0:
                return float(price)
            
            # 2. Try post-market price (after-hours)
            price = stock_data.get('postMarketPrice')
            if price and price > 0:
                return float(price)
            
            # 3. Try pre-market price (before open)
            price = stock_data.get('preMarketPrice')
            if price and price > 0:
                return float(price)
            
            # 4. Fallback to previous close (market closed)
            price = stock_data.get('regularMarketPreviousClose')
            if price and price > 0:
                return float(price)
    
    # 5. yfinance fallback...
```

**Improvements**:
✅ 4-tier fallback system
✅ After-hours support
✅ Pre-market support
✅ Previous close fallback
✅ Always returns a price (unless data unavailable)
