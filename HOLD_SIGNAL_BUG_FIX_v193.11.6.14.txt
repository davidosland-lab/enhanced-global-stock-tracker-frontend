🔍 ROOT CAUSE ANALYSIS
================================================================================

FILE: core/paper_trading_coordinator.py
LINE: 1335
METHOD: should_allow_trade()

BUGGY CODE:
-----------
```python
# Check signal action
prediction = signal.get('prediction', 0)
action = signal.get('action', '')

# Support both formats: prediction==1 OR action in ['BUY', 'STRONG_BUY']
is_buy_signal = (prediction == 1) or (action in ['BUY', 'STRONG_BUY'])

if not is_buy_signal:
    return True, 1.0, "Signal not a buy"  # ← BUG! Returns TRUE!
```

THE BUG:
--------
When `is_buy_signal` is False (meaning the signal is HOLD, SELL, or other):
  - Code returns: `True, 1.0, "Signal not a buy"`
  - True = Allow trade
  - 1.0 = Full position size
  - Result: HOLD/SELL signals are ALLOWED to trade!

INVERTED LOGIC:
---------------
The method should_allow_trade() returns:
  - Tuple[bool, float, str]
  - bool: True = ALLOW trade, False = BLOCK trade
  - float: Position multiplier (0.0 to 1.5+)
  - str: Reason message

When signal is NOT a BUY:
  ❌ Was returning: (True, 1.0, "Signal not a buy") → Allow with full size
  ✅ Should return: (False, 0.0, "Signal not actionable") → Block

HOW IT WAS USED:
----------------
In enter_position() method (line 1878):

```python
gate, position_multiplier, reason = self.should_allow_trade(...)

if not gate:  # If gate is False, block the trade
    logger.warning(f"{symbol}: TRADE BLOCKED - {reason}")
    return False
```

So when gate=True (from buggy code), the trade was ALLOWED!

✅ SOLUTION IMPLEMENTED
================================================================================

FIXED CODE:
-----------
```python
# Check signal action
prediction = signal.get('prediction', 0)
action = signal.get('action', '')

# Support both formats: prediction==1 OR action in ['BUY', 'STRONG_BUY']
is_buy_signal = (prediction == 1) or (action in ['BUY', 'STRONG_BUY'])

if not is_buy_signal:
    reason = f"Signal not actionable: {action if action else 'HOLD'} (only BUY/STRONG_BUY allowed)"
    logger.info(f"[BLOCK] {symbol}: {reason}")
    return False, 0.0, reason  # ← FIXED! Returns FALSE!
```

CHANGES:
--------
1. ✅ Changed: return True → return False
2. ✅ Changed: position 1.0 → 0.0 (no position when blocked)
3. ✅ Improved: Generic "Signal not a buy" → Descriptive reason with signal type
4. ✅ Added: [BLOCK] log prefix for visibility
5. ✅ Changed: logger.info → Appropriate for blocking non-actionable signals

LOGIC FLOW AFTER FIX:
---------------------
1. Check if signal is BUY or STRONG_BUY
2. If NOT → Return (False, 0.0, reason)
3. should_allow_trade returns False
4. enter_position() sees gate=False
5. Trade is BLOCKED with warning log
6. Stock is NOT added to positions
7. Only BUY/STRONG_BUY signals can proceed
