=============================================================================
  TROUBLESHOOTING
=============================================================================

PROBLEM: Slider still shows 65% after installation
SOLUTION:
  1. Clear browser cache (Ctrl+Shift+Delete)
  2. Hard refresh page (Ctrl+F5)
  3. Verify file core/unified_trading_dashboard.py has value=48
  4. Delete __pycache__: find . -name '__pycache__' -type d -exec rm -rf {} +
  5. Restart dashboard

PROBLEM: Trades still blocked at 65%
SOLUTION:
  1. Check dashboard slider position (move it to 48% if needed)
  2. Check log file: logs/paper_trading.log
  3. Look for "Using confidence threshold: XX.X%"
  4. If shows 65%, manually move slider to 48%
  5. Stop and restart trading

PROBLEM: Dashboard won't load
SOLUTION:
  1. Check port 8050 is not in use: netstat -an | findstr 8050
  2. Check Python version: python --version (need 3.8+)
  3. Reinstall dependencies: pip install -r requirements.txt
  4. Check firewall/antivirus isn't blocking port 8050

PROBLEM: "Module not found" errors
SOLUTION:
  1. Run install_complete.bat again
  2. Or manually: pip install -r requirements.txt
  3. Make sure you're in the correct directory
  4. Check Python path is correct

PROBLEM: No market data updating
SOLUTION:
  1. Check internet connection
  2. Verify yfinance is installed: pip show yfinance
  3. Check market hours (trading only during exchange hours)
  4. Review logs/paper_trading.log for errors

PROBLEM: FinBERT not loading
SOLUTION:
  1. Verify finbert_v4.4.4/ directory exists
  2. Check models/ subdirectory is present
  3. Review logs for import errors
  4. May need to reinstall transformers: pip install transformers
