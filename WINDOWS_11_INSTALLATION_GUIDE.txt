================================================================================
  UNDERSTANDING EVENT RISK GUARD
================================================================================

What It Does:
─────────────
Event Risk Guard protects you from event-driven losses by:

1. Detecting upcoming events 7 days in advance:
   - Basel III Pillar 3 reports (ASX banks)
   - Earnings announcements (±3 day sit-out windows)
   - Dividend ex-dates (±1 day sit-out windows)

2. Calculating risk scores (0-1 scale):
   - 0.0-0.25: Low risk (20% position haircut)
   - 0.25-0.50: Moderate risk (45% position haircut)
   - 0.50-0.80: High risk (70% position haircut)
   - 0.80-1.0: Extreme risk (SKIP - sit out)

3. Analyzing 72-hour sentiment:
   - FinBERT analysis on recent news headlines
   - Negative sentiment threshold: -0.10
   - Regulatory reports weighted 3.0x

4. Detecting volatility spikes:
   - 10-day vs 30-day rolling volatility
   - Spike threshold: 1.35x multiplier

5. Calculating rolling beta:
   - Beta vs ASX 200 (^AXJO)
   - Suggests hedge ratios for risk management


Real-World Example (CBA -6.6% Loss):
─────────────────────────────────────
Date: November 11, 2025
Event: CBA Basel III Pillar 3 Report released
Result: CBA.AX dropped -6.6% on the day

WITHOUT Event Risk Guard:
  - System saw positive news sentiment
  - Generated BUY signal
  - Lost -6.6% on position

WITH Event Risk Guard:
  - Detected Basel III report 7 days in advance
  - Analyzed 72h sentiment (mixed/negative)
  - Detected volatility spike
  - Risk score: 0.85 (extreme risk)
  - Action: SKIP TRADING (forced HOLD)
  - Result: LOSS AVOIDED

This is why Event Risk Guard was created.


Position Sizing Example:
────────────────────────
Stock: ANZ.AX
Event: Earnings announcement in 2 days
Risk Score: 0.55 (moderate-high risk)
Weight Haircut: 45%

Original target position: $10,000
Adjusted position: $10,000 × (1 - 0.45) = $5,500

If prediction is correct: +5% on $5,500 = +$275
If prediction is wrong: -5% on $5,500 = -$275

Without haircut, loss would be -$500 (nearly 2x worse)

================================================================================
  TROUBLESHOOTING
================================================================================

Issue 1: "Python is not recognized"
────────────────────────────────────
Problem: Python not in system PATH
Solution:
  1. Reinstall Python and CHECK "Add Python to PATH"
  2. Or manually add Python to PATH:
     - Search "Environment Variables" in Windows
     - Edit "Path" variable
     - Add: C:\Python311\ (or your Python installation path)


Issue 2: "TensorFlow not installed"
────────────────────────────────────
Problem: TensorFlow missing or not installed
Solution:
  1. Re-run INSTALL.bat
  2. Or install manually: pip install tensorflow>=2.13.0


Issue 3: "lxml version conflict"
─────────────────────────────────
Problem: yahooquery requires lxml<5.0.0
Solution: Already handled in INSTALL.bat
  - yahooquery is installed FIRST
  - This automatically installs correct lxml version
  - If you see warnings, ignore them


Issue 4: Training too slow (>30 min per stock)
───────────────────────────────────────────────
Problem: Large dataset, slow CPU, or insufficient RAM
Solution:
  1. Reduce epochs (edit screening_config.json):
     "epochs": 30  // Instead of 50
  
  2. Reduce batch size (if out of memory):
     "batch_size": 16  // Instead of 32
  
  3. Close other applications to free RAM


Issue 5: "Not enough data (XX days)"
─────────────────────────────────────
Problem: Stock has insufficient historical data
Solution:
  - Stock is too new or delisted
  - Skip this stock
  - Check if symbol is correct (e.g., CBA.AX not CBA)


Issue 6: Models not being used in pipeline
───────────────────────────────────────────
Problem: Pipeline not detecting trained models
Checklist:
  ✅ Models exist in models\lstm\ directory?
  ✅ Filenames match: {SYMBOL}_lstm_model.keras?
  ✅ Metadata files exist: {SYMBOL}_lstm_metadata.json?
  ✅ TensorFlow installed? (run VERIFY_INSTALLATION.bat)

Solution: Re-train models using TRAIN_LSTM_OVERNIGHT.bat


Issue 7: "ModuleNotFoundError"
───────────────────────────────
Problem: Python module not found
Solution: Re-run INSTALL.bat to install all dependencies


Issue 8: CSV export empty or missing columns
─────────────────────────────────────────────
Problem: Pipeline failed or incomplete
Solution:
  1. Check logs\screening\overnight_pipeline.log for errors
  2. Verify internet connection (for data fetching)
  3. Re-run RUN_OVERNIGHT_PIPELINE.bat

================================================================================
  FREQUENTLY ASKED QUESTIONS
================================================================================

Q1: Do I need to train LSTM models?
────────────────────────────────────
A: No, it's optional. System works in FinBERT-only mode without LSTM models.
   However, training improves accuracy by 7-12%.

Q2: How long does training take?
─────────────────────────────────
A: 10-15 minutes per stock, 1-2 hours for batch of 10 stocks.
   Run overnight or on weekends.

Q3: How often should I retrain models?
───────────────────────────────────────
A: Weekly recommended. Models older than 7 days are considered "stale".

Q4: Can I disable Event Risk Guard?
────────────────────────────────────
A: Yes, edit screening_config.json and set "event_risk": {"enabled": false}

Q5: What if event detection misses an event?
─────────────────────────────────────────────
A: Add it manually to models\config\event_calendar.csv

Q6: Can I use this for US stocks?
──────────────────────────────────
A: Yes, but Event Risk Guard is optimized for ASX banks (Basel III).
   Earnings and dividend detection works for all stocks.

Q7: Do I need a GPU for LSTM training?
───────────────────────────────────────
A: No, CPU training works fine. GPU would be faster but not required.

Q8: What if installation fails?
────────────────────────────────
A: Check internet connection, run as Administrator, or install manually:
   pip install -r requirements.txt

Q9: Can I change training parameters?
──────────────────────────────────────
A: Yes, edit models\config\screening_config.json
   Change epochs, batch_size, validation_split, etc.

Q10: Is this compatible with Windows 10?
─────────────────────────────────────────
A: Yes, fully compatible with Windows 10 and Windows Server 2019+.
