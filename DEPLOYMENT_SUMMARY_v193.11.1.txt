QUICK SUMMARY
===============================================================================
This is a hotfix release that resolves UnicodeEncodeError on Windows systems
when logging messages containing Unicode characters (arrows, emojis).

WHAT'S NEW IN v193.11.1
===============================================================================
✓ Fixed UnicodeEncodeError on Windows (cp1252 encoding)
✓ Added UTF-8 encoding to all logging handlers
✓ Replaced Unicode arrow (→) with ASCII (->)
✓ Replaced emoji (🚨) with ASCII ([!])
✓ Console output now works correctly on all platforms

CRITICAL FIX
===============================================================================
Issue:
------
System crashed with:
  UnicodeEncodeError: 'charmap' codec can't encode character '\u2192'
  
Location:
---------
- core/unified_trading_dashboard.py (line 1654)
- core/paper_trading_coordinator.py (line 2248)

Solution:
---------
1. Added encoding='utf-8' to logging.FileHandler
2. Changed StreamHandler() to StreamHandler(sys.stdout)
3. Replaced non-ASCII characters with ASCII equivalents
