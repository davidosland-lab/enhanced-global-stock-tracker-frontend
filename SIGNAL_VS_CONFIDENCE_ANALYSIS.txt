================================================================================
SIGNAL TYPE VS CONFIDENCE THRESHOLD - ANALYSIS & RECOMMENDATION
================================================================================
Version: v193.11.6.15
Date: 2026-03-09
Issue: HOLD signals with high confidence (>53%) are blocked from trading

================================================================================
PROBLEM SUMMARY
================================================================================

User Question: "Is rigidly sticking to the HOLD signal overriding the setting 
we had that trades could start at 53% confidence?"

Answer: YES - The current implementation has TWO separate filtering systems 
that work sequentially, and the signal-type filter comes FIRST (before 
confidence is checked).

================================================================================
CURRENT BEHAVIOR (v193.11.6.15)
================================================================================

Example from logs:
- CNA.L: Score 70.7/100, Signal HOLD, Confidence 53.6%
- BA.L: Score 60.7/100, Signal HOLD, Confidence 53.6%

Result: Both stocks BLOCKED from trading despite meeting 53% confidence threshold

================================================================================
TWO-STAGE FILTERING SYSTEM
================================================================================

STAGE 1: Signal Type Filter (HARD GATE)
----------------------------------------
Location: core/paper_trading_coordinator.py, line 612-645
Method: _evaluate_pipeline_recommendation()

Logic:
  if signal in ['BUY', 'STRONG_BUY']:
      # Process buy signals (check score, sentiment, confidence)
      confidence = (score * 0.7 + sentiment * 0.3)
      return True, confidence, reason
  
  elif signal in ['SELL', 'STRONG_SELL']:
      # Process sell signals
      return True, confidence, reason
  
  else:  # HOLD or any other signal
      return False, 0, f"Signal {signal} not actionable"

→ HOLD signals are REJECTED at this stage, before confidence is evaluated

STAGE 2: Confidence Filter
---------------------------
Location: core/paper_trading_coordinator.py, line 1340
Method: should_allow_trade()

Logic:
  min_confidence = 48.0  (or UI override, typically 53%)
  if confidence < min_confidence:
      return False, 0.0, f"Confidence {confidence:.1f}% < {min_confidence}%"

→ This stage is NEVER REACHED for HOLD signals (already rejected in Stage 1)

================================================================================
HOW HOLD SIGNALS ARE GENERATED
================================================================================

Location: pipelines/models/screening/batch_predictor.py
Method: _ensemble_prediction()

Logic:
  # Ensemble combines multiple ML models (LSTM, sentiment, technical, etc.)
  ensemble_direction = weighted average of model predictions
  
  # Threshold-based classification:
  if ensemble_direction > 0.3:
      prediction_label = 'BUY'
  elif ensemble_direction < -0.3:
      prediction_label = 'SELL'
  else:
      prediction_label = 'HOLD'  # Direction between -0.3 and +0.3

Key Point: 
- HOLD means "weak signal" or "neutral" - not strong bullish or bearish
- BUT confidence can still be HIGH (e.g., 53.6%)
- High confidence with HOLD = "System is confident the stock is neutral"

================================================================================
THE DESIGN QUESTION
================================================================================

Two Possible Interpretations:

Option A: Strict Signal-Type Gating (CURRENT)
----------------------------------------------
Philosophy: "Signal type is a directional prediction; confidence is certainty"
- HOLD = neutral prediction → Don't trade regardless of confidence
- BUY = bullish prediction → Trade if confidence ≥ threshold
- SELL = bearish prediction → Trade if confidence ≥ threshold

Pros:
✅ Respects ML model's directional assessment
✅ Avoids trading when system is uncertain about direction
✅ More conservative approach

Cons:
❌ Misses high-quality stocks with weak directional signals
❌ Confidence threshold becomes irrelevant for HOLD signals
❌ User's 53% threshold setting is bypassed

Option B: Confidence-Based Gating (PROPOSED)
---------------------------------------------
Philosophy: "If confidence is high enough, trade in the direction of bias"
- Score-based directional bias: Score > 50 = bullish bias
- If confidence ≥ threshold → Trade in direction of bias
- Ignore weak HOLD signals, use score as directional indicator

Pros:
✅ Respects user's confidence threshold setting
✅ Trades high-quality stocks (like CNA.L at 70.7)
✅ Confidence threshold has consistent meaning

Cons:
❌ May override ML model's neutral assessment
❌ Could trade when model says "don't trade"
❌ More aggressive approach

================================================================================
RECOMMENDED SOLUTION
================================================================================

Implement a HYBRID APPROACH with configuration option:

trade_mode: "strict" or "confidence_based"

Mode 1: Strict Signal-Type Gating (DEFAULT, current behavior)
-------------------------------------------------------------
- Only trade BUY/STRONG_BUY and SELL/STRONG_SELL signals
- HOLD signals always blocked
- Conservative, respects ML model

Mode 2: Confidence-Based with Score Bias
-----------------------------------------
- If confidence ≥ threshold:
    - Score ≥ 60 → Treat as BUY (even if signal is HOLD)
    - Score ≤ 40 → Treat as SELL (even if signal is HOLD)
    - 40 < Score < 60 → Respect HOLD signal (truly neutral)
- Allows trading high-quality HOLD signals

Implementation:

def _evaluate_pipeline_recommendation(self, recommendation: Dict) -> Tuple[bool, float, str]:
    """
    Evaluate if a pipeline recommendation meets trading signal parameters
    """
    symbol = recommendation['symbol']
    signal = recommendation['signal']
    score = recommendation['opportunity_score']
    sentiment = recommendation['sentiment']
    
    # Check configuration mode
    trade_mode = self.config.get('trade_mode', 'strict')  # Default to strict
    
    if trade_mode == 'strict':
        # CURRENT BEHAVIOR: Only trade explicit BUY/SELL signals
        if signal in ['BUY', 'STRONG_BUY']:
            # Check score, sentiment, age, calculate confidence
            if score < 60.0:
                return False, 0, f"Score too low: {score:.1f} < 60.0"
            if sentiment < 45.0:
                return False, 0, f"Sentiment too bearish: {sentiment:.1f} < 45.0"
            if recommendation['report_age_hours'] > 12:
                return False, 0, f"Report too old: {recommendation['report_age_hours']:.1f}h > 12h"
            
            confidence = (score * 0.7 + sentiment * 0.3)
            return True, confidence, f"Pipeline BUY: score={score:.1f}, sentiment={sentiment:.1f}"
        
        elif signal in ['SELL', 'STRONG_SELL']:
            if symbol not in self.positions:
                return False, 0, "No open position to sell"
            if score > 40.0:
                return False, 0, f"Score too high for sell: {score:.1f} > 40.0"
            
            confidence = (100 - score) * 0.7 + (100 - sentiment) * 0.3
            return True, confidence, f"Pipeline SELL: score={score:.1f}, sentiment={sentiment:.1f}"
        
        return False, 0, f"Signal {signal} not actionable (strict mode)"
    
    elif trade_mode == 'confidence_based':
        # NEW BEHAVIOR: Use confidence + score-based directional bias
        confidence = (score * 0.7 + sentiment * 0.3)
        
        # Check minimum confidence threshold
        min_confidence = self.config.get('min_confidence', 53.0)
        if confidence < min_confidence:
            return False, 0, f"Confidence {confidence:.1f}% < {min_confidence}%"
        
        # Check report age
        if recommendation['report_age_hours'] > 12:
            return False, 0, f"Report too old: {recommendation['report_age_hours']:.1f}h > 12h"
        
        # Determine trading direction based on score (override HOLD if score is strong)
        if signal in ['BUY', 'STRONG_BUY']:
            # Explicit buy signal - always trade if confidence met
            if score < 60.0:
                return False, 0, f"Score too low: {score:.1f} < 60.0"
            if sentiment < 45.0:
                return False, 0, f"Sentiment too bearish: {sentiment:.1f} < 45.0"
            return True, confidence, f"Pipeline BUY: score={score:.1f}, conf={confidence:.1f}%"
        
        elif signal in ['SELL', 'STRONG_SELL']:
            # Explicit sell signal
            if symbol not in self.positions:
                return False, 0, "No open position to sell"
            if score > 40.0:
                return False, 0, f"Score too high for sell: {score:.1f} > 40.0"
            return True, confidence, f"Pipeline SELL: score={score:.1f}, conf={confidence:.1f}%"
        
        elif signal == 'HOLD':
            # HOLD signal - use score-based bias
            if score >= 60.0 and sentiment >= 45.0:
                # High-quality HOLD → Treat as BUY
                return True, confidence, f"Pipeline BUY (HOLD override): score={score:.1f}, conf={confidence:.1f}%"
            
            elif score <= 40.0 and symbol in self.positions:
                # Low-quality HOLD + open position → Treat as SELL
                return True, confidence, f"Pipeline SELL (HOLD override): score={score:.1f}, conf={confidence:.1f}%"
            
            else:
                # Truly neutral HOLD (40 < score < 60)
                return False, 0, f"Signal HOLD with neutral score {score:.1f} (no bias)"
        
        return False, 0, f"Signal {signal} not actionable"
