INSTALLATION STEPS
================================================================================

STEP 1: Extract Package
------------------------
1. Extract unified_trading_system_v193.11.6.16_COMPLETE.zip
2. Extract to desired location (e.g., C:\Trading\)
3. Verify all files extracted successfully

STEP 2: Install Dependencies (FIRST TIME ONLY)
-----------------------------------------------
Windows:
  Double-click: INSTALL_FIRST_TIME.bat
  
Linux/Mac:
  chmod +x install_first_time.sh
  ./install_first_time.sh

This installs 35+ packages:
  - pandas, numpy, scipy
  - tensorflow, torch, transformers
  - dash, plotly, flask
  - yfinance, ta-lib (optional)
  - And more...

Wait for installation to complete (5-10 minutes)

STEP 3: Configure Trade Mode (OPTIONAL)
----------------------------------------
Default: "strict" mode (conservative, only BUY/SELL signals)

To enable confidence_based mode:

Option A - Via Dashboard (Easiest):
  1. Start dashboard (see Step 4)
  2. Find "Trade Mode" section
  3. Select "Confidence-Based (Allow high-confidence HOLD)"
  4. Start trading (config saved automatically)

Option B - Via config.json:
  1. Open: config/config.json
  2. Find "paper_trading" section:
     {
       "paper_trading": {
         "trade_mode": "confidence_based",  // Change this
         "min_confidence": 53.0,
         "hold_override_min_score": 60.0
       }
     }
  3. Save and close

STEP 4: Start Trading Dashboard
--------------------------------
Windows:
  Double-click: START_DASHBOARD.bat

Linux/Mac:
  chmod +x start_dashboard.sh
  ./start_dashboard.sh

Manual:
  cd core
  python unified_trading_dashboard.py

Dashboard opens at: http://localhost:8050

STEP 5: Load Stocks and Start Trading
--------------------------------------
In Dashboard:

1. Load stocks:
   - Manual: Enter symbols (comma-separated)
   - Auto: Click "Load Top 50 from Pipeline Reports"

2. Set parameters:
   - Initial Capital: e.g., 100000
   - Confidence Level: 48-95% (default 48%)
   - Stop Loss: 1-20% (default 10%)
   - Trade Mode: Strict or Confidence-Based

3. Click "Start Trading"

4. Monitor:
   - Open Positions
   - Trading History
   - Performance Metrics
   - ML Signal Components

TRADE MODE USAGE GUIDE
================================================================================

When to Use "strict" Mode (Default):
-------------------------------------
✓ Conservative trading approach
✓ Trust ML model's directional predictions
✓ Only want clear BUY/SELL signals
✓ Minimize false positives

Result: HOLD signals always blocked

When to Use "confidence_based" Mode:
-------------------------------------
✓ Want to trade high-quality stocks despite HOLD signal
✓ Confidence threshold (53%) should apply to ALL signals
✓ Willing to override ML model when fundamentals are strong
✓ Maximize trading opportunities

Result: HOLD signals with score≥60 & confidence≥53% traded as BUY

Example Decision Tree:
----------------------
Stock: CNA.L, Score: 70.7, Signal: HOLD, Confidence: 53.6%

Strict Mode:
  1. Check signal type → HOLD → BLOCK ❌
  (Confidence never checked)

Confidence_Based Mode:
  1. Check confidence → 53.6% ≥ 53% → PASS ✓
  2. Check signal type → HOLD
  3. Check score → 70.7 ≥ 60 → PASS ✓
  4. Check sentiment → (assume ≥45) → PASS ✓
  5. Result: TRADE AS BUY ✅
  Log: "[HOLD→BUY] CNA.L: Overriding HOLD signal - score=70.7, conf=53.6%"

================================================================================
TROUBLESHOOTING
================================================================================

Issue: "ModuleNotFoundError: No module named 'pandas'"
-------------------------------------------------------
Solution: Run INSTALL_FIRST_TIME.bat again
  - Ensure you have Python 3.8+ installed
  - Check pip is working: pip --version
  - Install manually: pip install -r requirements.txt

Issue: Dashboard won't start - "Address already in use"
--------------------------------------------------------
Solution: Port 8050 is in use
  - Find process: netstat -ano | findstr 8050
  - Kill process: taskkill /PID <pid> /F
  - Or change port in unified_trading_dashboard.py (line ~2500)

Issue: HOLD signals still blocked in confidence_based mode
-----------------------------------------------------------
Solution: Check configuration
  1. Verify config.json: "trade_mode": "confidence_based"
  2. Check logs for "[HOLD→BUY]" messages (not "[BLOCK]")
  3. Verify score≥60 and confidence≥53 for the HOLD signal
  4. Check sentiment≥45 (another requirement)

Issue: Too many HOLD signals being traded
------------------------------------------
Solution: Adjust thresholds
  1. Increase hold_override_min_score: 60.0 → 65.0 or 70.0
  2. Increase min_confidence: 53.0 → 60.0 or 65.0
  3. Or switch back to "strict" mode

Issue: "UnicodeEncodeError" in logs
------------------------------------
Solution: Should be fixed in v193.11.6.15
  - All emoji replaced with ASCII
  - If still occurs, check for custom log formatters
  - Ensure UTF-8 encoding in Python: sys.stdout.reconfigure(encoding='utf-8')

Issue: US pipeline "AttributeError: 'EventRiskGuard' object has no attribute 'get_latest_regime'"
--------------------------------------------------------------------------------------------------
Solution: Should be fixed in v193.11.6.13
  - Ensure you're using v193.11.6.13 or later
  - Check VERSION.json shows correct version
  - Re-extract deployment package if needed

Issue: Gap predictions missing in reports
------------------------------------------
Solution: Check pipeline execution
  1. Verify overnight pipeline ran successfully
  2. Check reports/screening/ for latest *.json files
  3. Ensure spx_proxy_realtime.py (US), spi_proxy_realtime.py (AU) exist
  4. Re-run pipeline with --mode full flag

Issue: Sentiment scores too high during market sell-offs
---------------------------------------------------------
Solution: Should be fixed in v193.11.6.12
  - Sentiment now incorporates gap prediction data
  - Check logs for "Sentiment Adjusted for Gap Prediction" messages
  - Verify gap_adjusted_sentiment=True in market_sentiment dict
